## 一、系统定位

**半醒之间 个人IP系统** 是一人AI自媒体操作系统，
面向"一个普通人在AI时代探索个人价值重构"的方向，用 AI Agent 替代传统自媒体团队岗位，
实现从选题、研究、创作、发布、复盘到商业变现的完整闭环。

### 来源
本系统由 ChatGPT 导师对话（15轮定位梳理）驱动搭建，以 Claude Code 作为系统工程管理器。

## 七、Workflow 自动化流程（5个）

### 7.1 内容生产流程
> 文件：[[workflows/content_production_workflow]]

```
输入：一个主题或灵感
    │
    ▼
 0. 内容层定位     ← 判断属于觉察/重构/创造哪一层
    │
    ▼
 1. 用户分析       ← 研究Agent + 用户痛点挖掘Skill
    │
    ▼
 2. 选题优化       ← 选题Agent + 赛道分析Skill + 三层校准
    │
    ▼
 3. 爆款参考       ← 爆款分析Agent + 爆款拆解Skill
    │
    ▼
 4. 生成标题       ← 内容Agent + 标题生成Skill
    │
    ▼
 5. 生成正文       ← 内容Agent + 文案写作Skill（含三层语气适配）
    │
    ▼
 6. 视觉设计       ← 视觉Agent + 封面设计/AI绘图Skill（品牌视觉对齐）
    │
    ▼
 7. 保存归档       ← 知识库Agent → 05_内容生产库/（含三层标签）
    │
    ▼
 8. 发布小红书     ← Creator执行（手动审核）
    │
    ▼
 9. 数据追踪       ← 复盘Agent + 数据分析Skill → 10_数据复盘/
    │
    ▼
    复盘报告 → 进入下一轮迭代
```

### 7.2 用户研究工作流
> 文件：[[workflows/research_workflow]]
- **频率**：每周2次
- **流程**：评论扫描 → 搜索热词采集 → 三层分类 → 用户洞察卡片 → 更新Memory
- **产出**：评论分析报告、用户洞察卡片、按三层分类的新选题建议

### 7.3 数据复盘工作流
> 文件：[[workflows/review_workflow]]
- **频率**：每周日执行
- **流程**：数据整理 → 三层效果对比 → 90天实验进度检查 → 策略输出 → 归档
- **产出**：周度复盘报告、内容评分卡、下周计划（含三层比例调整）

### 7.4 爆款研究工作流
> 文件：[[workflows/viral_analysis_workflow]]
- **频率**：每周2次
- **流程**：爆款发现（半醒之间视角）→ 深度拆解 → 模型提炼 → 归档更新
- **产出**：爆款拆解报告、爆款模型卡、可迁移选题

### 7.5 商业变现工作流
> 文件：[[workflows/business_workflow]]
- **频率**：每月1次 + 按需触发
- **当前阶段**：⚪ 监控模式（仅记录洞察，不推动变现）
- **流程**：机会评估 → 产品设计 → 转化路径 → 上线执行 → 复盘优化
- **产出**：产品设计方案、转化漏斗、推广计划

## 八、设计原则

| 原则 | 说明 |
|------|------|
| Agent负责目标 | 每个Agent有明确的角色、目标和产出 |
| Skill负责能力 | 每个Skill是可独立执行的标准化能力单元 |
| Memory负责长期资产 | 记忆系统积累可持续复用的知识资产 |
| Task负责执行 | 工具完成具体操作，Agent调度工具 |
| Workflow负责流程 | 工作流串联Agent和Skill形成完整闭环 |
| 三层意识 | 所有内容先判断觉察/重构/创造层级，按层调整语气和策略 |

> 🚫 不设计万能Agent：每个Agent职责边界清晰，通过协作而非全能完成复杂任务。
