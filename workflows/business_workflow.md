# 商业变现工作流（半醒之间版）

> 自动化商业机会评估和变现流程
> 关联定位：[[01_个人定位/半醒之间IP定位.md|半醒之间IP定位]]
> ⚠️ **当前阶段（0-10000 粉）**：工作流处于**监控模式**——只记录洞察，不推动变现。
> 当前唯一商业任务：**先成为案例**。

---

```yaml
预估耗时: 30-45 分钟
current_phase: "监控模式"  # 阶段: 监控模式 → 探索模式 → 执行模式
```

## 流程步骤

### Phase 1: 机会评估（10分钟）
**执行 Agent**: [[agents/business_agent.md|Business Agent]]
**调用 Skill**: [[skills/monetization/产品设计.md|产品设计]]

1. **用户需求扫描**
   - 分析评论区高频需求
   - 统计私信中付费意愿
   - 调研竞品付费产品

2. **需求验证**
   - 需求强度评估（普遍性/紧迫性）
   - 付费意愿评估
   - 已有竞品分析

3. **能力匹配**
   - 评估个人专业知识是否支撑
   - 确定所需资源和时间
   - 预估投入产出比

### Phase 2: 产品设计（15分钟）
**执行 Agent**: [[agents/business_agent.md|Business Agent]]

1. **产品概念定义**
   - 核心价值定位
   - 目标用户确认
   - 差异化卖点

2. **产品形态设计**
   - 交付形式（课程/社群/咨询/电子书）
   - 内容大纲
   - 交付流程设计

3. **定价策略**
   - 价值定价评估
   - 竞品价格参考
   - 定价方案（标准/进阶/尊享）

### Phase 3: 转化路径设计（10分钟）
**执行 Agent**: [[agents/business_agent.md|Business Agent]]
**调用 Skill**: [[skills/monetization/私域运营.md|私域运营]]

1. **导流方案**
   - 诱饵设计（免费资料/体验课）
   - 导流路径规划
   - 话术设计

2. **承接流程**
   - 加好友后自动回复
   - 信任建立内容序列
   - 转化时机设计

3. **推广计划**
   - 预热内容规划
   - 发布渠道选择
   - 节奏安排

### Phase 4: 上线执行
**执行 Agent**: Creator（人类）+ [[agents/business_agent.md|Business Agent]]

1. **内容预热**
   - 发布预告内容
   - 营造期待感
   - 收集预报名

2. **正式发布**
   - 全渠道推送
   - 1对1核心用户触达
   - 限时优惠/早鸟价

3. **交付执行**
   - 按计划完成交付
   - 收集用户反馈
   - 迭代优化

### Phase 5: 复盘优化（10分钟）
**执行 Agent**: [[agents/business_agent.md|Business Agent]] + [[agents/review_agent.md|Review Agent]]

1. **数据复盘**
   - 销量/收入/成本
   - 用户满意度
   - 转化率分析

2. **经验沉淀**
   - 成功因素总结
   - 失败教训记录
   - 迭代计划

3. **产品线更新**
   - 更新产品矩阵
   - 调整定价策略
   - 规划下一代产品

## 商业日历

### 月度商业节奏
```
第1周：评估新机会 + 现有产品优化
第2周：产品开发/迭代
第3周：预热内容发布
第4周：正式推广 + 复盘
```

### 季度商业节奏
```
第1个月：产品开发期
第2个月：推广爆发期
第3个月：服务交付 + 复盘迭代
```
