# 爆款研究工作流（半醒之间版）

> 自动化爆款发现、拆解和模型提炼流程
> 分析视角：以「半醒之间」的"普通人AI时代价值重构"视角审视爆款
> 适用频率：每周执行1-2次

---

```yaml
name: 爆款研究工作流（半醒之间版）
trigger: 定时触发 / 发现爆款时触发
频率: 每周2次

```

## 流程步骤

### Phase 1: 爆款发现（5分钟）
**执行Agent**: 爆款分析Agent

1. **领域内扫描**
   - 搜索个人成长领域高互动笔记
   - 记录近期（7天内）数据异常的笔记

2. **跨领域参考**
   - 关注其他领域的爆款模式
   - 评估是否可迁移到个人成长领域

3. **新作者监控**
   - 发现快速增长的新账号
   - 分析其增长策略

### Phase 2: 深度拆解（15分钟）
**执行Agent**: 爆款分析Agent
**调用Skill**: skills/data/爆款拆解.md

1. **标题拆解**
   - 还原标题公式
   - 分析关键词策略
   - 评估情绪触发点

2. **内容结构分析**
   - 骨架还原（SCQA/故事/清单等）
   - 节奏设计分析（段落长度、信息密度）
   - 情绪曲线还原

3. **开头钩子分析**
   - 前3句话拆解
   - 钩子类型归类
   - 为什么有效

4. **互动设计分析**
   - 评论区置顶策略
   - 互动引导话术
   - 争议点设计

### Phase 3: 模型提炼（5分钟）
**执行Agent**: 爆款分析Agent

1. **共性提取**
   - 与之前爆款的共同特征
   - 领域特有的爆款公式

2. **模型命名**
   - 为提炼的爆款模型命名
   - 明确适用条件和变量

3. **可迁移性评估**
   - 评估模型是否适合个人品牌
   - 确定需要调整的部分

### Phase 4: 归档输出（5分钟）
**执行Agent**: 知识库Agent

1. **创建爆款拆解报告**
   - 填写爆款拆解模板
   - 保存到 04_爆款拆解库/

2. **更新爆款模型库**
   - 新增或更新模型
   - 关联相关选题和内容

3. **通知相关Agent**
   - 通知选题Agent可借鉴的选题
   - 通知内容Agent可复用的结构

## 爆款选择标准

| 条件 | 要求 | 优先级 |
|------|------|--------|
| 阅读量 | > 10,000 | ⭐⭐⭐ |
| 互动率 | > 10% | ⭐⭐⭐ |
| 相关性 | 与个人成长领域相关 → **优先"普通人视角+时代困惑"型** | ⭐⭐⭐ |
| 时效性 | 发布不超过30天 | ⭐⭐ |
| 可迁移性 | 可适配「半醒之间」定位 | ⭐⭐⭐ |
| 三层适用性 | 可归入觉察/重构/创造某一层 | ⭐⭐ |
| 多样性 | 不同类型的爆款 | ⭐ |
