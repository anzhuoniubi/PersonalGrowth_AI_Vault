    # 系统自检与优化提升 — 提示词

> 位置：07_提示词库/
> 用途：触发CEO Agent执行全链路健康检查
> 使用方法：将下方提示词直接发给Claude即可

---

## 完整提示词

```
请对 Personal Growth AI Self Media OS 执行一次完整的全链路系统自检。

## 检查范围

### 1. 🧠 Agent健康检查
逐一检查9个Agent配置文件的完整性，确认：
- 每个Agent的Role/Goal/Workflow/Skills是否定义完整
- 是否有Agent的配置与实际Skill脱节
- 是否有Agent长期未被调用（检查memory/选题/内容库中的引用记录）
- 输出：Agent健康评分 + 需要更新的Agent列表

### 2. 🔧 Skill覆盖度检查
检查 20 个基础 Skill 的实际使用情况：
- 哪些 Skill 从未在内容生产流程中被调用
- 哪些 Skill 缺少实际案例/模板（只有框架没有内容）
- 相比 10 个 Vault 数字目录，是否有场景缺少对应的 Skill
- 重点关注：数据类（3个）、研究类（5个）是否被实际使用
- 输出：Skill热力地图 + 缺失Skill建议

### 3. 🔄 Workflow执行检查
检查5个Workflow的实际跑通情况：
- content_production_workflow — 是否每次出内容都走了完整流程
- review_workflow — 上次复盘是什么时候
- research_workflow — 用户研究是否有周期性执行
- viral_analysis_workflow — 爆款拆解是否在持续
- business_workflow — 商业化进度
- 输出：各Workflow最后执行时间 + 流程断点

### 4. 🧬 Memory系统检查
检查4个长期记忆的状态：
- brand_memory — 品牌定位是否清晰，有无迭代
- audience_memory — 用户画像是否过时
- knowledge_memory — 知识体系是否在持续积累
- content_style — 风格指南是否需要更新
- 输出：记忆新鲜度评分 + 需要更新的记忆

### 5. 📂 Vault目录利用率
检查 10 个 Vault 数字目录的使用情况：
- 列出每个目录的文件数和最近更新时间
- 标记空置目录（0文件）和低利用率目录
- 检查目录命名与实际内容是否一致
- 输出：目录热力图 + 清理/合并建议

### 6. 📊 数据复盘状态
拉取所有已发布内容的复盘进度：
- 哪些内容已发布但未做72小时复盘
- 哪些内容有数据但未更新到选题卡片
- 互动率Top/Bottom的内容分析
- 输出：待复盘清单 + 数据洞察

### 7. 🏗️ 结构合规检查（新增）
- 每个 Agent 是否包含 11 个标准段（标题/引用/Role/Goal/Workflow/Skills/Tools/Memory/输出格式/关键指标/三层对齐/页脚）
- 每个方法型 Skill 是否包含六段式（使用场景/输入格式/执行流程/输出格式/示例/页脚）
- 每个 Workflow 是否使用单一编号（无重复 Step/Phase）
- 全量 wiki 链接是否存在断链
- 输出：结构合规评分 + 待修复清单

## 输出格式
以看板形式输出，每个板块用emoji标记状态：
🟢 健康 / 🟡 警示 / 🔴 需修复 / ⚪ 未涉及

每个红色/黄色项目附上具体的修复步骤（可执行、不废话）。

## 最后
基于以上检查，输出Top 3优化行动项（按优先级排序），
并建议由哪个Agent来执行。
```

---

## 使用场景

| 频率 | 场景 | 动作 |
|------|------|------|
| **每周一** | 系统周检 | 跑一遍，看过去一周系统健康度 |
| **内容数据异常** | 排查原因 | 检查是否某个环节断连 |
| **每月初** | 月度优化 | 基于检查结果安排本月优化任务 |
| **新成员/新Skill加入后** | 集成检查 | 确认新组件融入系统 |
