## 三、各 MCP 详解

### 1. Sequential Thinking — 复杂推理

**解决什么问题：**
一次性跳到结论容易遗漏关键因素。Sequential Thinking 强制分步骤推演，每一步可回溯、可修正。

**调用方式：**
```
当任务满足以下任一条件时，优先使用：
- 涉及 ≥3 个变量的因果分析
- 需要在多个方案间做权衡
- 数据异常需要根因诊断
- 策略规划需要评估多个路径
```

**典型用法：**

| 场景 | 触发词 | 推演步骤 |
|------|--------|---------|
| 选题分析 | "这个选题为什么火/不火" | ① 提取选题特征 → ② 对照爆款模型 → ③ 归因到具体维度 → ④ 输出优化建议 |
| 数据诊断 | "XX篇为什么推流停了" | ① 检查CTR → ② 检查话题敏感度 → ③ 对比同类内容 → ④ 判断根因 |
| 策略规划 | "下阶段内容方向" | ① 回顾当前数据 → ② 评估三层比例 → ③ 列出候选方向 → ④ 按风险/收益排序 |

**调用 Agent：** 选题Agent、复盘Agent、CEO Agent

### 2. Playwright — 浏览器自动化

**解决什么问题：**
以往采集小红书数据只能手动复制或依赖导出Excel。Playwright 可以直接打开网页、截图、抓取结构化数据。

**调用方式：**
```
当任务涉及以下任一操作时使用：
- 打开小红书网页查看笔记、评论区
- 截图保存作为证据/参考
- 抓取搜索结果页的数据
- 自动化检查（如检查发布后的展示效果）
```

**典型用法：**

| 场景 | 操作 | 输出 |
|------|------|------|
| 评论区采集 | 打开笔记页 → 滚动评论区 → 提取评论文本 | 结构化评论数据 |
| 竞品扫描 | 搜索关键词 → 提取Top 10笔记标题、点赞数 | 竞品分析表 |
| 封面检查 | 打开个人主页 → 截图 → 检查封面一致性 | 视觉审计 |
| 搜索热词 | 搜索框输入 → 抓取下拉词 | 关键词列表 |

**调用 Agent：** 研究Agent、爆款分析Agent

**注意：**
- 不用于批量刷量（违反小红书规则）
- 每次请求间隔 ≥3 秒
- 仅采集公开可见数据

### 3. Filesystem — 文件管理

**解决什么问题：**
替代简单Bash命令，提供更结构化的文件读写能力。可批量操作、跨目录搜索。

**调用方式：**
```
当任务涉及以下操作时优先使用：
- 批量搜索多个目录中的内容
- 移动/重命名文件
- 读取目录结构
- 检查文件是否存在
```

**典型用法：**

| 场景 | 操作 |
|------|------|
| 知识库整理 | 扫描 `00_Inbox/` → 按分类移入对应目录 |
| 选题查重 | 搜索 `03_选题库/` 中是否有相同主题 |
| 断链检查 | 扫描所有 `.md` 文件中的 `[[link]]` → 验证目标是否存在 |
| 目录健康检查 | 统计各目录文件数，标记空置目录 |

**已配置的访问目录：**
- `/Users/mac/Documents/Obsidian/PersonalGrowth_AI_Vault`（主工作区）
- `/Users/mac/Downloads`（读取导出的 Excel 数据）

**调用 Agent：** 知识库Agent、内容Agent

### 5. Memory — 知识图谱

**解决什么问题：**
当前 Memory 系统是4个 Markdown 文件，只能被动阅读无法主动查询。
Memory MCP 构建知识图谱，支持实体-关系查询和跨会话记忆。

**存储位置：** `memory/mcp_knowledge_graph.json`

**典型用法：**

| 场景 | 存入 | 查询 |
|------|------|------|
| 用户画像更新 | 实体：用户画像 → 属性：痛点/场景/情绪词 | "当前用户最常提的3个痛点" |
| 选题模式沉淀 | 实体：选题 → 关系：属于→爆款模型 | "CTR>15%的选题有什么共性" |
| 内容复盘 | 实体：内容 → 属性：CTR/互动率/评级 | "互动率最低的3篇内容" |
| 经验积累 | 实体：教训 → 关系：发生在→Agent | "内容Agent踩过哪些坑" |

**写入原则：**
- 每条记忆必须可验证（有数据来源或日志支撑）
- 不写入完整正文内容（太大，适合文件引用）
- 每完成一轮会话后，写入关键经验

**查询原则：**
- 新会话开始时，先查询知识图谱加载上下文
- 遇到问题时，先查询是否有相关经验记录

**调用 Agent：** CEO Agent、研究Agent、复盘Agent

## 五、故障排查

| 问题 | 可能原因 | 修复方式 |
|------|---------|---------|
| MCP server 无法启动 | `node_modules` 未安装 | `npm install` |
| GitHub MCP 无权限 | Token 过期 | 重新生成 Token → 更新 `.claude/mcp.json` |
| Playwright 无法启动 | Chromium 未安装 | `npx playwright install chromium` |
| Memory 文件丢失 | 首次使用前被删除 | 重新创建 `memory/mcp_knowledge_graph.json`（空文件） |
