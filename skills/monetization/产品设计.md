# 产品设计

## 使用场景

- 知识付费产品规划
- 变现产品线设计
- 内容产品化转型
- 用户需求 → 产品方案

## 输入格式

### 必须输入
| 字段 | 类型 | 说明 |
|------|------|------|
| user_needs | object[] | 痛点+付费意愿 |
| creator_expertise | string[] | 创作者专业能力 |
| product_type | string | course / consulting / community / digital_product |

### 可选输入
| 字段 | 类型 | 说明 |
|------|------|------|
| current_assets.audience_size | number | 当前粉丝数 |
| current_assets.engagement_rate | number | 互动率 |
| business_goal.revenue_target | number | 收入目标 |
| business_goal.timeline | string | 时间周期 |

## 执行流程

1. **需求验证**：用户是否愿意为这个需求付费
2. **能力匹配**：我的专业知识是否足够支撑
3. **产品定义**：MVP 最小可行产品设计
4. **价值提炼**：用户获得的具体价值
5. **定价策略**：价值定价 vs 成本定价 vs 竞品定价
6. **交付设计**：交付形式和流程
7. **产品线规划**：引流品 → 利润品 → 高端品

### 产品线模型
- **引流品**（免费/低价）：PDF、清单、模板 → 建立信任
- **标准品**（中价）：课程、训练营 → 交付价值
- **高端品**（高价）：1 对 1 咨询、私教 → 深度服务

### 判断标准（if-then）
- 如果没有 10 条以上用户付费意愿证据，则禁止进入开发。
- 如果定价低于用户月收入的 1%，则重新评估价值感知。
- 如果产品承诺结果无法验证，则必须调整产品边界。

## 示例

### 示例 1：知识体系训练营
**输入**：
```yaml
user_needs:
  - pain_point: "学了很多记不住，知识零散不成体系"
    willingness_to_pay: "high"
creator_expertise: ["知识管理", "Obsidian", "个人成长"]
product_type: "course"
current_assets:
  audience_size: 5000
  engagement_rate: 0.08
business_goal:
  revenue_target: 10000
  timeline: "30天"
```

**输出**：
```markdown
## 产品设计方案
### 产品定位
- 产品名称：21 天个人知识体系搭建训练营
- 解决痛点：学了很多记不住，知识零散不成体系
- 目标用户：25-35 岁想建立个人知识体系的职场人
- 核心价值：一套可复用的知识管理方法 + 实操

### 产品详情
- 形式：社群 + 录播 + 作业 + 直播答疑
- 定价：699 元
- 交付周期：21 天
- 内容大纲：Week1 认知篇 - Week2 工具篇 - Week3 实战篇

### 产品线位置
- 产品类型：标准品
- 转化路径：免费内容 → 低价 PDF → 训练营 → 1 对 1 咨询

### 风险提示
- 可能问题：完课率低
- 应对方案：每日轻量作业 + 社群陪伴 + 前 3 天重点运营
```
