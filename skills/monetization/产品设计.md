# 产品设计 Skill

## 1. Skill名称
产品设计

## 2. 使用场景
- 知识付费产品规划
- 变现产品线设计
- 内容产品化转型
- 用户需求→产品方案

## 3. 输入格式
```yaml
user_needs:
  - pain_point: ""
    willingness_to_pay: "high" | "medium" | "low"
creator_expertise: []
current_assets:
  - content_series: []
  - audience_size: 0
  - engagement_rate: 0
business_goal:
  revenue_target: 0
  timeline: ""
product_type: "course" | "consulting" | "community" | "digital_product"
```

## 4. 执行流程
1. **需求验证**：用户是否愿意为这个需求付费
2. **能力匹配**：我的专业知识是否足够支撑
3. **产品定义**：MVP最小可行产品设计
4. **价值提炼**：用户获得的具体价值
5. **定价策略**：价值定价 vs 成本定价 vs 竞品定价
6. **交付设计**：交付形式和流程
7. **产品线规划**：引流品→利润品→高端品

### 产品线模型
- **引流品**（免费/低价）：PDF、清单、模板 → 建立信任
- **标准品**（中价）：课程、训练营 → 交付价值
- **高端品**（高价）：1对1咨询、私教 → 深度服务

## 6. 示例
```markdown
## 产品设计方案
### 产品定位
- 产品名称："21天个人知识体系搭建训练营"
- 解决痛点：学了很多记不住，知识零散不成体系
- 目标用户：25-35岁想建立个人知识体系的职场人
- 核心价值：一套可复用的知识管理方法+实操

### 产品详情
- 形式：社群+录播+作业+直播答疑
- 定价：699元
- 交付周期：21天
- 内容大纲：Week1 认知篇 - Week2 工具篇 - Week3 实战篇

### 产品线位置
- 产品类型：标准品
- 转化路径：免费内容→低价PDF→训练营→1对1咨询
```
