# 课程设计

## 使用场景

- 设计知识付费课程体系（免费课 → 低价课 → 高价课）
- 将零散的内容经验打包成结构化课程
- 规划课程的学习路径和交付节奏
- 评估课程的市场可行性和定价

## 输入格式

### 必须输入
| 字段 | 类型 | 说明 |
|------|------|------|
| course_topic | string | 课程主题 |
| target_audience | string | 目标用户 |
| course_type | string | 训练营 / 录播课 / 社群课 / 电子书 / 模板包 |
| learner_goal | string | 用户学完能获得什么 |

### 可选输入
| 字段 | 类型 | 说明 |
|------|------|------|
| price_range | string | free / low(<100) / medium(100-500) / high(>500) |
| current_content_base | string[] | 已有的相关笔记/内容列表 |

## 执行流程

1. **需求验证（A）**
   - 从评论区/私信找出用户反复问的 3-5 个问题
   - 搜索竞品，看已有课程的评价差评（找到差异化空间）
   - 回答：粉丝真的愿意为这个主题付费吗？

2. **课程设计（D）**
   - 输出 1 页课程大纲（3 章 × 3 节起）
   - 确定交付形式和时间周期
   - 定价参考公式：**同学费 = 用户月收入的 5-10%**
     - 目标用户月收入 15k-40k → 课程定价 750-4000 元可行
     - 首次建议从 299-599 元起步验证

3. **内容开发（D）**
   - 把已有的笔记/内容按课程结构重组
   - 每个章节补充"配得上的干货量"
   - 制作交付物（PPT/模板/作业/案例库）

4. **上线推广（I）**
   - 预热期：发 3 条与课程主题相关的内容
   - 转化期：限时优惠/早鸟价
   - 交付期：按节奏交付，前 3 天重点运营

5. **评估迭代（E）**
   - 收集完课率和满意度
   - 记录用户提问（这些就是下一期的优化方向）
   - 高满意度 → 出进阶版
   - 低满意度 → 分析是内容问题还是预期错配

### 核心框架（流程内小节）

#### 框架 A：ADDIE 模型
> 来源：美国佛罗里达州立大学（1975年）
> 适用：系统化课程设计，确保不缺步骤

```
A（Analysis）— 分析
    ↓ 用户需求分析 → 竞品课程调研 → 可行性判断
D（Design）— 设计
    ↓ 学习目标设定 → 课程大纲 → 交付形式 → 定价
D（Development）— 开发
    ↓ 内容制作 → 视觉设计 → 平台搭建
I（Implementation）— 实施
    ↓ 推广预热 → 开课 → 社群运营
E（Evaluation）— 评估
    ↓ 完课率 → 满意度 → 复购意愿 → 迭代
```

#### 框架 B：课程金字塔（三层产品线）
```
                    ▲ 高客单价·低数量
                   / \
      私教/1对1    /高 \   → 2000-5000元
                 /  交 \
                /  付   \
      训练营    │ 中客单 │  → 299-999元
               │ 中数量 │
               │  标准品  │
              /│         │\
  引流课/     │ 低客单   │  → 免费-99元
  电子书/     │  高数量  │
  模板包      │  引流品   │
             ─────────────
```

| 层 | 形式 | 定价 | 目标 | 关键指标 |
|---|------|:----:|------|---------|
| 引流品 | PDF模板/清单/电子书/免费直播 | 免费-29.9 | 获取信任+沉淀私域 | 领取人数/私域转化率 |
| 标准品 | 训练营/录播课/社群 | 299-999 | 规模收入+品牌沉淀 | 报名人数/完课率 |
| 高端品 | 1对1咨询/私教陪跑 | 2000-5000 | 深度服务+高利润 | 交付时长/NPS |

> 原则：没有引流品就不要推标准品，没有标准品就不要推高端品。

#### 框架 C：学习路径设计的 4 步节奏
> 来源：Gagne 九大教学事件 + 知识付费交付实践

```
第一性：我从哪里来
    ↓ 共情+痛点唤醒（让用户觉得"这就是我"）
    ↓ 占课程前 10% 时间

第二性：我要去哪里
    ↓ 明确学习目标（让用户知道学完能怎样）
    ↓ 占课程前 5% 时间

第三性：我怎么去
    ↓ 核心内容交付（最小知识量+最大可执行性）
    ↓ 占课程 70% 时间

第四性：我怎么知道到了
    ↓ 作业/实践+反馈（完成闭环）
    ↓ 占课程 15% 时间
```

### 判断标准（if-then）
- 如果完课率 < 60%，则检查内容难度/作业量/社群运营是否匹配。
- 如果满意度 < 4.5/5，则优先分析差评是内容问题还是预期错配。
- 如果没有 10 条以上用户付费意愿证据，则禁止进入开发阶段。

## 示例

### 示例 1：AI 时代个人品牌课
**输入**：
```yaml
course_topic: "普通人的AI时代个人品牌搭建指南"
target_audience: "25-35岁职场人，被传统路径压抑但相信还有可能"
course_type: "录播课+社群"
price_range: "medium"
current_content_base: ["AI时代个人价值重构系列"]
learner_goal: "建立一套从自我觉察到个人品牌的完整成长系统"
```

**输出**：
```markdown
## 课程方案

### 基本信息
- 课程名称：普通人 AI 时代个人品牌·12 周系统搭建
- 目标用户：25-35 岁职场人，被传统路径压抑但相信还有可能
- 定价：399 元
- 形式：录播课（6 节）+ 社群答疑
- 周期：21 天

### 课程大纲
第1章：理解你的内耗（Week 1）
  第1节：内耗的底层机制 — 不是你的错
  第2节：你的内耗是你的本金 — 认知反转
  第3节：从消耗到存入 — 捕捉系统搭建

第2章：让内耗产生复利（Week 2）
  第1节：连接的艺术 — 让笔记产生化学反应
  第2节：临界点突破 — 从 100 条到 200 条的质变
  第3节：情绪波动时的系统驾驶指南

### 定价与销售
| 阶段 | 价格 | 权益 |
|------|:----:|------|
| 早鸟价（前 20 人） | 299 元 | 课程+社群+1 次作业点评 |
| 正价 | 399 元 | 课程+社群 |
| 老学员价 | 199 元 | 仅课程（不含社群） |

### 推广计划
- 预热期：发 3 篇相关主题内容（铺垫）
- 推广期：课程介绍+早鸟名额
- 开课：21 天社群陪伴交付
```

### 避坑清单
- [ ] 不要先做课再验证需求 — 先在评论区/私信测试付费意愿
- [ ] 不要追求"完整" — 第一次做 4-6 节即可，MVP 先跑
- [ ] 不要定价过低 — 9.9 元吸引来的用户不珍惜，完课率低
- [ ] 不要忽略社群运营 — 课程内容只占 50%，社群陪伴是另外 50%
- [ ] 不要一次性全量交付 — 分期交付才能保持完课率和社群活跃
- [ ] 不要承诺超出交付能力的结果 — "学完月入过万"→踩红线
- [ ] 不要忽视老学员 — 满意学员的推荐是成本最低的获客渠道
