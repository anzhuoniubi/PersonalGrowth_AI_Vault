# 课程设计 Skill

> 验证来源：ADDIE模型 / SAM模型 / 知识付费行业实践（小鹅通/知识星球头部案例）
> 最后更新：2026-07-17

---

## 1. Skill名称
课程设计

## 2. 使用场景
- 设计知识付费课程体系（免费课→低价课→高价课）
- 将零散的内容经验打包成结构化课程
- 规划课程的学习路径和交付节奏
- 评估课程的市场可行性和定价

## 3. 输入格式
```yaml
course_topic: ""
target_audience: ""
course_type: "训练营" | "录播课" | "社群课" | "电子书" | "模板包"
price_range: "free" | "low" (<100) | "medium" (100-500) | "high" (>500)
current_content_base: []  # 已有的相关笔记/内容列表
learner_goal: ""  # 用户学完能获得什么
```

## 4. 核心框架

### 框架A：ADDIE模型（课程设计流程）

> 来源：美国佛罗里达州立大学（1975年）
> 适用：系统化课程设计，确保不缺步骤

```
A（Analysis）— 分析
    ↓ 用户需求分析 → 竞品课程调研 → 可行性判断
D（Design）— 设计
    ↓ 学习目标设定 → 课程大纲 → 交付形式 → 定价
D（Development）— 开发
    ↓ 内容制作 → 视觉设计 → 平台搭建
I（Implementation）— 实施
    ↓ 推广预热 → 开课 → 社群运营
E（Evaluation）— 评估
    ↓ 完课率 → 满意度 → 复购意愿 → 迭代
```

**每步具体做什么：**

```
A—分析（1-2周）
  □ 从粉丝评论和私信中验证是否有付费意愿
  □ 搜索竞品课程，分析定价和差评痛点
  □ 结论：做还是不做

D—设计（1周）
  □ 输出课程大纲（至少3章×3节）
  □ 确定交付形式（直播/录播/图文/社群）
  □ 定价参考：粉丝量×0.5-1元（1万粉→5k-1万定价）

D—开发（2-4周）
  □ 逐节内容制作
  □ 每节课输出"讲稿+PPT/图文+作业"
  □ MVP（最小可行版本）可先上架再迭代

I—实施
  □ 预热：提前1周发3条相关内容做铺垫
  □ 开课：按节奏交付，前3天最关键（决定留存）
  □ 社群：每天固定时间互动

E—评估
  □ 完课率 > 60% → 合格
  □ 满意度 > 4.5/5 → 可迭代为高价版
  □ 差评分类：内容问题/交付问题/预期错配
```

### 框架B：课程金字塔（三层产品线）

> 来源：知识付费行业通用模型
> 适用：设计从引流到高客单的课程矩阵

```
                    ▲ 高客单价·低数量
                   / \
      私教/1对1    /高 \   → 2000-5000元
                 /  交 \
                /  付   \
      训练营    │ 中客单 │  → 299-999元
               │  中数量 │
               │  标准品  │
              /│         │\
  引流课/     │ 低客单   │  → 免费-99元
  电子书/     │  高数量  │
  模板包      │  引流品   │
             ─────────────
```

| 层 | 形式 | 定价 | 目标 | 关键指标 |
|---|------|:----:|------|---------|
| 引流品 | PDF模板/清单/电子书/免费直播 | 免费-29.9 | 获取信任+沉淀私域 | 领取人数/私域转化率 |
| 标准品 | 训练营/录播课/社群 | 299-999 | 规模收入+品牌沉淀 | 报名人数/完课率 |
| 高端品 | 1对1咨询/私教陪跑 | 2000-5000 | 深度服务+高利润 | 交付时长/NPS |

> 原则：没有引流品就不要推标准品，没有标准品就不要推高端品。

### 框架C：学习路径设计的4步节奏

> 来源：Gagne九大教学事件 + 知识付费交付实践
> 适用：设计单节课/单模块的学习体验

```
第一性：我从哪里来
    ↓ 共情+痛点唤醒（让用户觉得"这就是我"）
    ↓ 占课程前10%时间
    ↓ 示例："你是不是也收藏了很多方法但从未执行？"

第二性：我要去哪里
    ↓ 明确学习目标（让用户知道学完能怎样）
    ↓ 占课程前5%时间
    ↓ 示例："今天学完，你会得到一张可以直接用的清单"

第三性：我怎么去
    ↓ 核心内容交付（最小知识量+最大可执行性）
    ↓ 占课程70%时间
    ↓ 示例：分步骤讲解+案例+模板

第四性：我怎么知道到了
    ↓ 作业/实践+反馈（完成闭环）
    ↓ 占课程15%时间
    ↓ 示例：布置一个5分钟能完成的作业+点评
```

## 5. 执行流程

### Step 1：需求验证（A）
- 从评论区/私信找出用户反复问的3-5个问题
- 搜索竞品，看已有课程的评价差评（找到差异化空间）
- 回答：粉丝真的愿意为这个主题付费吗？

### Step 2：课程设计（D）
- 输出1页课程大纲（3章×3节起）
- 确定交付形式和时间周期
- 定价参考公式：**同学费 = 用户月收入的5-10%**
  - 目标用户月收入15k-40k → 课程定价750-4000元可行
  - 首次建议从299-599元起步验证

### Step 3：内容开发（D）
- 把已有的笔记/内容按课程结构重组
- 每个章节补充"配得上的干货量"
- 制作交付物（PPT/模板/作业/案例库）

### Step 4：上线推广（I）
- 预热期：发3条与课程主题相关的内容
- 转化期：限时优惠/早鸟价
- 交付期：按节奏交付，前3天重点运营

### Step 5：评估迭代（E）
- 收集完课率和满意度
- 记录用户提问（这些就是下一期的优化方向）
- 高满意度 → 出进阶版
- 低满意度 → 分析是内容问题还是预期错配

## 8. 避坑清单

- [ ] **不要先做课再验证需求** — 先在评论区/私信测试付费意愿
- [ ] **不要追求"完整"** — 第一次做4-6节即可，MVP先跑
- [ ] **不要定价过低** — 9.9元吸引来的用户不珍惜，完课率低
- [ ] **不要忽略社群运营** — 课程内容只占50%，社群陪伴是另外50%
- [ ] **不要一次性全量交付** — 分期交付才能保持完课率和社群活跃
- [ ] **不要承诺超出交付能力的结果** — "学完月入过万"→踩红线
- [ ] **不要忽视老学员** — 满意学员的推荐是成本最低的获客渠道
