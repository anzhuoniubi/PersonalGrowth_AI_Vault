# A/B测试 Skill

## 1. Skill名称
A/B测试

## 2. 使用场景
- 标题效果对比测试
- 封面方案效果验证
- 发布时间优化
- 内容格式选择
- 互动策略验证

## 3. 输入格式
```yaml
test_variable: "title" | "cover" | "publish_time" | "format" | "hook_style"
variant_a: {}
variant_b: {}
control_group: {}
success_metric: "click_rate" | "interaction_rate" | "save_rate"
sample_size: 0
test_duration: ""
```

## 4. 执行流程
1. **假设提出**：明确要测试的变量和预期
2. **变量控制**：只改变一个变量，其他保持一致
3. **版本设计**：设计A/B两个版本
4. **测试实施**：在不同时间或不同渠道发布
5. **数据收集**：收集两个版本的表现数据
6. **显著性检验**：判断差异是否统计显著
7. **结论输出**：确定优胜版本和原因
8. **知识沉淀**：将结论纳入创作规范

### 测试原则
- **单一变量**：一次只测一个变量
- **足够样本**：至少1000次曝光再下结论
- **同时测试**：尽量在同一时段测试
- **完整周期**：覆盖完整的内容生命周期
- **记录存档**：所有测试结果留档

## 5. 输出格式
```markdown
## A/B测试报告
### 测试信息
- 测试变量：{}
- 测试时间：{}
- 样本量：{}

### 结果对比
| 指标 | 版本A | 版本B | 差异 |
|------|-------|-------|------|
| 点击率 | {}% | {}% | {}% |
| 互动率 | {}% | {}% | {}% |
| 收藏率 | {}% | {}% | {}% |

### 结论
- 优胜版本：{}
- 显著程度：{高/中/低}
- 原因分析：{}

### 行动建议
- 留存实践：{}
- 下次优化方向：{}
```

## 6. 示例
```markdown
## A/B测试报告
### 测试信息
- 测试变量：标题公式
- 版本A：数字型"3个方法提升学习效率"
- 版本B：疑问型"为什么你的学习效率一直提不高？"
- 测试时间：7天

### 结果对比
| 指标 | 版本A | 版本B | 差异 |
|------|-------|-------|------|
| 点击率 | 8.2% | 5.1% | +3.1% |
| 互动率 | 6.5% | 4.2% | +2.3% |

### 结论
- 优胜版本：版本A（数字型）
- 显著程度：高
- 原因分析：数字型标题更具体，给读者明确的预期

### 行动建议
- 后续内容优先使用数字型标题
- 测试数字型的不同数字效果
```
