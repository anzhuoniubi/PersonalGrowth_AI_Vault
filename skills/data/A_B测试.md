# A/B 测试

## 使用场景

- 标题效果对比测试
- 封面方案效果验证
- 发布时间优化
- 内容格式选择
- 互动策略验证

## 输入格式

### 必须输入
| 字段 | 类型 | 说明 |
|------|------|------|
| test_variable | string | title / cover / publish_time / format / hook_style |
| variant_a | object | A 版本 |
| variant_b | object | B 版本 |
| success_metric | string | click_rate / interaction_rate / save_rate |

### 可选输入
| 字段 | 类型 | 说明 |
|------|------|------|
| control_group | object | 对照组 |
| sample_size | number | 样本量 |
| test_duration | string | 测试时长 |

## 执行流程

1. **假设提出**：明确要测试的变量和预期
2. **变量控制**：只改变一个变量，其他保持一致
3. **版本设计**：设计 A/B 两个版本
4. **测试实施**：在不同时间或不同渠道发布
5. **数据收集**：收集两个版本的表现数据
6. **显著性检验**：判断差异是否统计显著
7. **结论输出**：确定优胜版本和原因
8. **知识沉淀**：将结论纳入创作规范

### 测试原则
- **单一变量**：一次只测一个变量
- **足够样本**：至少 1000 次曝光再下结论
- **同时测试**：尽量在同一时段测试
- **完整周期**：覆盖完整的内容生命周期
- **记录存档**：所有测试结果留档

### 判断标准（if-then）
- 如果样本量 <1000，则标注"样本不足，结论仅供参考"。
- 如果差异 <5%，则标记为"无显著差异"。
- 如果优胜版本与假设相反，则必须复盘变量控制是否有效。

## 输出格式

### 必须字段
| 字段 | 类型 | 说明 |
|------|------|------|
| 测试变量 | string | — |
| 测试时间 | string | — |
| 样本量 | number | — |
| 结果对比 | object | 各指标 A/B 对比 |
| 优胜版本 | string | A/B/无显著差异 |
| 显著程度 | string | 高/中/低 |
| 行动建议 | string[] | — |

## 示例

### 示例 1：标题公式测试
**输入**：
```yaml
test_variable: "title"
variant_a: "3个方法提升学习效率"
variant_b: "为什么你的学习效率一直提不高？"
success_metric: "click_rate"
sample_size: 2000
test_duration: "7天"
```

**输出**：
```markdown
## A/B 测试报告
### 测试信息
- 测试变量：标题公式
- 版本 A：数字型"3个方法提升学习效率"
- 版本 B：疑问型"为什么你的学习效率一直提不高？"
- 测试时间：7 天
- 样本量：2000

### 结果对比
| 指标 | 版本 A | 版本 B | 差异 |
|------|-------|-------|------|
| 点击率 | 8.2% | 5.1% | +3.1% |
| 互动率 | 6.5% | 4.2% | +2.3% |

### 结论
- 优胜版本：版本 A（数字型）
- 显著程度：高
- 原因分析：数字型标题更具体，给读者明确的预期

### 行动建议
- 后续内容优先使用数字型标题
- 测试数字型的不同数字效果
```
