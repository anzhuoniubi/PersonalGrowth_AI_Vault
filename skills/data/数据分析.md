# 数据分析 Skill

## 1. Skill名称
数据分析（半醒之间版）

## 2. 使用场景
- 每周数据复盘 → 含三层内容维度对比
- 内容效果评估 → 按觉察/重构/创造分层分析
- 增长归因分析
- 90天实验计划进度追踪
- 策略效果量化

## 3. 输入格式
```yaml
posts_data:
  - title: ""
    publish_date: ""
    views: 0
    likes: 0
    favorites: 0
    comments: 0
    shares: 0
    followers_gained: 0
    content_layer: "觉察层" | "重构层" | "创造层"  # 三层标注
time_range: "weekly" | "monthly" | "quarterly"
comparison: "previous_period" | "same_period_last_month"
```

## 3.5 三层分析维度（新增）
每次数据分析增加以下维度：
1. **各层效果对比**：觉察层/重构层/创造层的平均互动率、收藏率、评论率
2. **比例偏差分析**：当前发布比例是否偏离计划？哪一层需要补充？
3. **阶段适应性**：当前数据是否支持进入90天实验的下一阶段？

## 4. 执行流程
1. **数据清洗**：整理和校验原始数据
2. **核心指标计算**：互动率、转化率、增长率
3. **维度分析**：时间/内容类型/格式多维度
4. **基准对比**：对比历史数据和行业基准
5. **异常识别**：找到表现特别好/差的内容
6. **趋势分析**：识别增长/下降趋势
7. **归因分析**：什么因素驱动了变化
8. **洞察提炼**：转化为可执行的建议

### 核心指标
- **互动率** = (点赞+收藏+评论+分享) / 阅读量
- **转化率** = 粉丝增长 / 阅读量
- **爆款率** = 阅读量>1万的内容占比
- **粉丝质量** = 互动粉丝 / 总粉丝
- **内容效率** = 总互动 / 创作时长

## 5. 输出格式
```markdown
## 数据分析报告
### 核心数据
| 指标 | 本期 | 上期 | 变化 |
|------|------|------|------|
| 总阅读 | {} | {} | {}% |
| 总互动 | {} | {} | {}% |
| 互动率 | {} | {} | {}% |
| 粉丝增长 | {} | {} | {}% |

### 维度分析
- 最佳内容类型：{}
- 最佳发布时间：{}
- 最佳标题公式：{}

### 异常发现
- 高光内容：{原因分析}
- 低效内容：{原因分析}

### 策略建议
1. {建议}
2. {建议}
```

## 6. 示例
```markdown
## 数据分析报告
### 核心数据
| 指标 | 本期 | 上期 | 变化 |
|------|------|------|------|
| 总阅读 | 52,300 | 38,100 | +37% |
| 总互动 | 4,180 | 2,850 | +47% |
| 互动率 | 8.0% | 7.5% | +0.5% |
| 粉丝增长 | 320 | 210 | +52% |

### 维度分析
- 最佳内容类型：方法论型（互动率12%）
- 最佳发布时间：周四20:00-21:00
- 最佳标题公式：数字+结果型

### 策略建议
1. 增加方法论型内容比例至60%
2. 优化故事型内容的开头钩子
```
