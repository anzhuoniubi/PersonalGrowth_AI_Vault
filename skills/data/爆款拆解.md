# 爆款拆解

## 使用场景

- 发现爆款内容时的深度分析
- 建立爆款模型库
- 内容创作前的参考研究
- 规律总结和方法论提炼

## 输入格式

### 必须输入
| 字段 | 类型 | 说明 |
|------|------|------|
| target_post.title | string | 爆款标题 |
| target_post.url | string | 链接 |
| target_post.author | string | 作者 |
| target_post.metrics | object | 阅读/点赞/收藏/评论/分享 |

### 可选输入
| 字段 | 类型 | 说明 |
|------|------|------|
| analysis_depth | string | quick / deep |
| content_type | string | 图文 / 视频 |
| niche | string | 领域 |

## 执行流程

1. **标题拆解**：分析标题公式和钩子
2. **封面分析**：视觉吸引力和信息传达
3. **开头拆解**：前 3 句话如何抓住注意力
4. **结构分析**：内容骨架和节奏
5. **情绪曲线**：读者情绪起伏设计
6. **共鸣点识别**：哪些点引发评论互动
7. **金句提取**：传播性强的句子
8. **模型归纳**：提炼可复用的爆款模型

### 分析维度
- **选题角度**：为什么这个话题会爆？
- **表达方式**：如何让内容有感染力？
- **互动设计**：如何引导用户评论？
- **传播基因**：如何激发转发和收藏？
- **时机因素**：是否踩中热点或趋势？

### 判断标准（if-then）
- 如果阅读 <1 万且互动率 <10%，则不进入深度拆解。
- 如果爆款逻辑与「半醒之间」定位冲突，则标注"不可迁移"。
- 如果发布时间超过 30 天，则标注"时效性待验证"。

## 输出格式

### 必须字段
| 字段 | 类型 | 说明 |
|------|------|------|
| 基础信息 | object | 标题/数据/作者/时间 |
| 拆解分析 | object | 标题/开头/结构/情绪/互动 |
| 爆款模型 | string | 可复用公式 |
| 我的应用 | string | 如何应用到自己的内容 |
| 三层归属 | string | 觉察/重构/创造 |
| 可迁移性 | string | 高/中/低 |

## 示例

### 示例 1：年龄+数字公式拆解
**输入**：
```yaml
target_post:
  title: "30岁才明白的5个人生道理"
  url: "https://..."
  author: "XXX"
  metrics:
    views: 50000
    likes: 5200
    favorites: 1800
    comments: 300
analysis_depth: "deep"
```

**输出**：
```markdown
## 爆款拆解报告
### 基础信息
- 标题：30岁才明白的5个人生道理
- 数据：5.2w 赞，1.8w 收藏
- 作者：XXX
- 发布时间：周四晚

### 拆解分析
| 维度 | 分析 | 借鉴 |
|------|------|------|
| 标题 | 年龄标签+数字+人生感悟 | 替换为个人职业视角 |
| 开头 | 直接抛出第一个道理，不铺垫 | 前 3 秒必须给价值 |
| 结构 | 5 个并列道理，每个配一个小故事 | 适合清单型内容 |
| 情绪 | 温暖+轻微后悔+希望 | 觉察层共鸣 |
| 互动 | 结尾问"你明白了吗" | 改为开放式问题 |

### 爆款模型
"年龄+数字+人生感悟"公式
- 年龄标签引发精准人群共鸣
- 数字增加预期和可读性
- 人生感悟满足"过来人经验"需求

### 我的应用
创作"28岁产品经理的3个认知升级"
保留年龄+数字结构，替换为个人职业视角

### 可迁移性
高（结构通用，只需替换身份标签）
```
