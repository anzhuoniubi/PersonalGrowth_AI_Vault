# 用户需求建模

## 使用场景

- 从用户评论、私信、调研中提取结构化的需求模型
- 判断用户需求的真伪与优先级
- 将模糊的用户反馈转化为可执行的内容选题或产品设计输入

## 输入格式

### 必须输入
| 字段 | 类型 | 说明 |
|------|------|------|
| raw_data.source | string | 评论/私信/调研/社群聊天 |
| raw_data.sample_size | number | 样本数量 |
| raw_data.excerpts | string[] | 原文摘录 |
| target_output | string | 选题输入 / 产品设计输入 / 内容优化 |

## 执行流程

1. **素材采集**：收集至少 10 条用户原话，记录原文，不做加工
2. **JTBD 翻译**：对每条用户原话用 JTBD 框架翻译，找出背后的"Jobs"
3. **三层归因**：对每个 Job 做三层需求穿透，找到身份层需求
4. **KANO 优先级**：将所有需求按 KANO 模型分类，标记必备/期望/魅力/反向
5. **输出需求卡片**：每个高优先级需求输出一张需求卡片

### 核心框架（流程内小节）

#### 框架 A：JTBD（Jobs to Be Done）
> 来源：Clayton Christensen《创新者的窘境》
> 核心理念：用户不是购买产品/内容，而是"雇佣"它来完成一个任务。

```
用户说的一句话                     → 背后的 Job
"我就是执行力太差"                → 帮我克服开始行动的阻力
"收藏了但从来没看过"              → 帮我在需要时检索到内容
"又内耗了一整天什么都没干"        → 帮我中断反刍循环
```

**三步走：**
1. 记录用户的原话（不加工）
2. 问：用户雇佣这个内容/产品，想完成什么"任务"？
3. 写下一个完整的句子：**当___时，我想___，以便___。**

#### 框架 B：KANO 模型
> 来源：狩野纪昭（Noriaki Kano）

| 属性 | 定义 | 内容示例 |
|------|------|---------|
| ✅ 必备属性 | 不做会被骂，做了用户不觉得特别 | 标题与封面一致、正文不标题党 |
| 📈 期望属性 | 做得越好用户越满意 | 信息密度、可执行性、干货量 |
| 🎁 魅力属性 | 做了让用户惊喜，不做也无所谓 | 金句、情绪共鸣、个人故事 |
| ❌ 反向属性 | 做了反而让用户不开心 | 广告感太强、说教语气 |

**决策原则：**
- 先确保必备属性（不被骂）
- 再提升期望属性（提升互动率）
- 最后用魅力属性制造惊喜（促进转发和收藏）

#### 框架 C：三层需求归因模型
```
第一层：表面需求（用户说出来的）
    ↓ 问"为什么重要？"
第二层：深层需求（用户真正想要的）
    ↓ 问"这能带来什么感觉？"
第三层：身份需求（用户想成为谁）
```

### 判断标准（if-then）
- 如果只有 1-2 条相似表达，则标记为"个案，不做模式"。
- 如果需求被归类为反向属性，则必须排除。
- 如果身份层需求与品牌定位冲突，则必须重新评估。

## 输出格式

### 必须字段
| 字段 | 类型 | 说明 |
|------|------|------|
| 需求编号 | string | — |
| 用户原话 | string | 禁止编造 |
| JTBD | string | 当…我想…以便… |
| 三层归因 | object | 表面/深层/身份 |
| KANO 属性 | string | 必备/期望/魅力/反向 |
| 优先级 | string | 高/中/低 |
| 内容启发 | string | — |

### 输出示例
```markdown
## 需求卡片

### 需求编号：{N}

用户原话：{}
─────────────────────────
JTBD：当{}，我想{}，以便{}。
─────────────────────────
三层归因：
  表面：{}
  深层：{}
  身份：{}
─────────────────────────
KANO属性：{必备/期望/魅力}
优先级：{高/中/低}
─────────────────────────
内容启发：{}
```

## 示例

### 示例 1：自律焦虑需求
**输入**：
```yaml
raw_data:
  source: "小红书评论区"
  sample_size: 15
  excerpts:
    - "每次看完那些自律博主我就更焦虑了"
    - "不是不想努力，是不知道从哪里开始"
    - "为什么别人能坚持我不能"
target_output: "选题输入"
```

**输出**：
```markdown
## 需求卡片

### 需求编号：001

用户原话："每次看完自律博主我就更焦虑了"
─────────────────────────
JTBD：当我在深夜看到别人在努力时，我想获得"不努力也可以"的许可，
      以便我能摆脱"别人在进步我在原地"的焦虑。
─────────────────────────
三层归因：
  表面：讨厌被自律内容刺激
  深层：想要被接纳当下的状态
  身份：想成为一个"即使不完美也值得被爱"的人
─────────────────────────
KANO属性：魅力属性（做到了就是爆款）
优先级：高
─────────────────────────
内容启发："别再'向内求'了"这篇正是从"理解你的不努力"切入，
而非"教你努力"，精准命中了这个需求。
```

### 避坑清单
- [ ] 不把用户的表面表达当真实需求
- [ ] 不做过度解读——10 条不同用户的相似表达才算一个模式
- [ ] 不忽略反向属性——有些需求做了反而掉粉
- [ ] 需求卡片要定期回顾（每月至少一次）
- [ ] JTBD 分析结果要和内容策略对齐
