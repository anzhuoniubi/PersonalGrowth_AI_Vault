# 情绪分析

## 使用场景

- 内容情感基调设计
- 评论区情绪监测
- 选题情绪匹配
- 品牌情感定位

## 输入格式

### 必须输入
| 字段 | 类型 | 说明 |
|------|------|------|
| text_content | string | 待分析的文本 |
| analysis_type | string | comment / content / topic |

### 可选输入
| 字段 | 类型 | 说明 |
|------|------|------|
| focus_dimensions | string[] | intensity / valence / trigger_words |
| context | string | 分析背景 |

## 执行流程

1. **情绪识别**：识别文本中的情绪标签
2. **强度评估**：情绪强烈程度（1-10）
3. **正负判断**：积极/消极/中性
4. **触发词提取**：引发情绪的关键词
5. **情绪归因**：情绪背后的原因
6. **内容适配**：根据情绪设计内容基调
7. **共鸣点匹配**：找到情绪与内容的连接点

### 判断标准（if-then）
- 如果情绪强度 ≥8 且为负面，则必须标注"高敏感情绪，回复需谨慎"。
- 如果负面占比 ≥50%，则必须触发危机预警并上报 CEO Agent。
- 如果触发词与品牌定位冲突，则必须标注"品牌风险"。

## 输出格式

### 必须字段
| 字段 | 类型 | 说明 |
|------|------|------|
| 主导情绪 | string | — |
| 情绪强度 | number | 1-10 |
| 情绪基调 | string | 积极/消极/中性 |
| 触发词 | string[] | — |
| 内容策略 | object | 匹配情绪/建议语气/避免雷区/共鸣点 |

### 输出示例
```markdown
## 情绪分析结果
### 情绪画像
- 主导情绪：{}
- 情绪强度：{}
- 情绪基调：{积极/消极/中性}

### 触发因素
- 触发词：{}
- 情绪来源：{}

### 内容策略
- 匹配情绪：{}
- 建议语气：{}
- 避免雷区：{}
- 情感共鸣点：{}
```

## 示例

### 示例 1：评论区焦虑情绪
**输入**：
```yaml
text_content: "30岁还一事无成，同龄人都有房有车了，我该怎么办"
analysis_type: "comment"
focus_dimensions: ["intensity", "valence", "trigger_words"]
```

**输出**：
```markdown
## 情绪分析结果
### 情绪画像
- 主导情绪：焦虑 + 希望（混合）
- 情绪强度：7/10
- 情绪基调：积极焦虑（想变好但不知如何开始）

### 触发因素
- 触发词：30岁、瓶颈、落后、同龄人
- 情绪来源：社会比较 + 年龄焦虑

### 内容策略
- 匹配情绪：先用共情化解焦虑，再给解决方案
- 建议语气：温和坚定，不制造更多焦虑
- 避免雷区：不要用"再不xxx就晚了"的恐吓式表达
- 情感共鸣点：承认焦虑是正常的，给出可控的路径
```
