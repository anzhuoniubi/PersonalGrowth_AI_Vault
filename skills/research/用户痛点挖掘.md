# 用户痛点挖掘 Skill

## 1. Skill名称
用户痛点挖掘

## 2. 使用场景
- 选题发现阶段的痛点识别
- 内容创作前的用户需求确认
- 新产品/服务设计
- 用户洞察报告制作

## 3. 输入格式
```yaml
source_material:
  - type: "comment" | "dm" | "review" | "forum"
    content: ""
    platform: ""
    source_url: ""
target_audience: ""
focus_area: ""
```

## 4. 执行流程
1. **素材收集**：整理目标用户的真实表达
2. **表层问题提取**：用户说了什么
3. **深层需求归因**：用户真正需要什么
4. **情绪强度评估**：这个问题对用户有多痛
5. **频率统计**：多少个用户在说同样的问题
6. **场景还原**：在什么场景下这个问题最突出
7. **痛点分级**：核心痛点 vs 痒点 vs 伪痛点
8. **内容转化建议**：如何针对痛点创作

## 5. 输出格式
```markdown
## 痛点卡片
### 痛点：{一句话描述}
- 用户原话："{用户真实表达}"
- 表层需求：{用户以为需要的}
- 深层需求：{用户真正需要的}
- 情绪强度：{高/中/低}
- 出现频率：{常见/偶尔/罕见}
- 典型场景：{什么情况下最痛}
- 内容切入角度：{如何转化成内容}
```

## 6. 示例
```markdown
## 痛点卡片
### 痛点：看了很多书但记不住也用不上
- 用户原话："每年读50本书，但读完就忘，感觉白读了"
- 表层需求：更好的记忆方法
- 深层需求：需要一个将知识转化为能力的系统
- 情绪强度：高
- 出现频率：非常常见
- 典型场景：年底复盘时、读书社群讨论时
- 内容切入角度："如何构建个人知识体系"系列
```
