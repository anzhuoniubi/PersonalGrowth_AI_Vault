# 用户痛点挖掘

## 使用场景

- 选题发现阶段的痛点识别
- 内容创作前的用户需求确认
- 新产品/服务设计
- 用户洞察报告制作

## 输入格式

### 必须输入
| 字段 | 类型 | 说明 |
|------|------|------|
| source_material | object[] | 用户真实表达素材 |
| target_audience | string | 目标用户群体 |

### 可选输入
| 字段 | 类型 | 说明 |
|------|------|------|
| focus_area | string | 聚焦领域 |

### source_material 字段
| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | comment / dm / review / forum |
| content | string | 原文 |
| platform | string | 来源平台 |
| source_url | string | 来源链接 |

## 执行流程

1. **素材收集**：整理目标用户的真实表达
2. **表层问题提取**：用户说了什么
3. **深层需求归因**：用户真正需要什么
4. **情绪强度评估**：这个问题对用户有多痛
5. **频率统计**：多少个用户在说同样的问题
6. **场景还原**：在什么场景下这个问题最突出
7. **痛点分级**：核心痛点 vs 痒点 vs 伪痛点
8. **内容转化建议**：如何针对痛点创作

### 判断标准（if-then）
- 如果同一痛点在 10 条以上素材中出现，则标记为"核心痛点"。
- 如果情绪强度 ≥8 且频率 ≥50%，则标记为"高优先级"。
- 如果痛点无法通过内容解决，则标记为"伪痛点"或"需产品解决"。

## 输出格式

### 必须字段
| 字段 | 类型 | 说明 |
|------|------|------|
| 痛点 | string | 一句话描述 |
| 用户原话 | string | 禁止编造 |
| 表层需求 | string | — |
| 深层需求 | string | — |
| 情绪强度 | string | 高/中/低 |
| 出现频率 | string | 常见/偶尔/罕见 |
| 典型场景 | string | — |
| 内容切入角度 | string | — |

### 输出示例
```markdown
## 痛点卡片
### 痛点：{}
- 用户原话："{}"
- 表层需求：{}
- 深层需求：{}
- 情绪强度：{高/中/低}
- 出现频率：{常见/偶尔/罕见}
- 典型场景：{}
- 内容切入角度：{}
```

## 示例

### 示例 1：知识焦虑痛点
**输入**：
```yaml
source_material:
  - type: "comment"
    content: "每年读50本书，但读完就忘，感觉白读了"
    platform: "小红书"
  - type: "comment"
    content: "收藏了很多干货，但从来没打开过"
    platform: "小红书"
target_audience: "25-35岁职场人"
focus_area: "个人成长"
```

**输出**：
```markdown
## 痛点卡片
### 痛点：看了很多书但记不住也用不上
- 用户原话："每年读50本书，但读完就忘，感觉白读了"
- 表层需求：更好的记忆方法
- 深层需求：需要一个将知识转化为能力的系统
- 情绪强度：高
- 出现频率：非常常见
- 典型场景：年底复盘时、读书社群讨论时
- 内容切入角度："如何构建个人知识体系"系列
```
