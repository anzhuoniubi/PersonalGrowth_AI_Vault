# 近期爆火模板雷达：从发现词到可复用拍摄语法

> 用于回答“最近大家都在拍什么”“有哪些爆火拍摄/内容模板”“像从夯到拉这种模板怎么找”。
> 目标不是保存一张会过期的热梗名单，而是持续发现、验证、分期并调用模板家族。

## 0. 先纠正一个前提

“某篇很火”不等于“出现了可复用模板”。模板至少要有可被不同创作者重复的语法：相似的开场、镜头动作、剪辑节奏、文案槽位或参与规则。单篇高赞、同一账号的系列栏目、某个滤镜或一张梗图都只能先进入 `query_candidate`。

本雷达研究的是三件不同的事：

1. **当前供给**：在本次入口、时间窗和账号环境中，哪些结构正在被重复使用。
2. **公开代理表现**：点赞、收藏、评论、参与作业等可见信号是否与该结构共同出现。
3. **可迁移语法**：去掉作者、原句、原素材后，还剩下什么可以合法改编。

始终保留：`public_proxy ≠ traffic`。没有自有账号 impressions/reach，就不能把搜索位置或公开互动写成“平台给流量”“验证有效”或“必爆”。

## 1. 何时运行

出现以下意图时，必须先跑本雷达，再讨论选题或成稿：

- 近期爆火模板、拍摄灵感、全网都在拍、最近刷屏、跟拍、挑战、转场、热梗、互动图；
- 想找“从夯到拉”一类可套用结构；
- 想知道某个模板还在上升、已经成熟，还是已经拍烂；
- 要把热门模板沉淀到长期库，供以后生成时调用。

模式选择：

- 已有模板库或上次雷达：主模式用 `refresh`，从 `last_refreshed_at` 之后复查。
- 没有模板库、类目也不确定：主模式用 `discovery`，先宽搜再建家族。
- 已知某个模板，只想判断能不能跟：仍用窄范围 `refresh`，不能直接进入 `draft`。

所有 v2 run 都必须显式填写 `trend_template_requirement: none|research|draft`。`discovery/refresh` 真正在搜模板时填 `research`；成稿确实调用候选卡时填 `draft`；不用模板也必须显式填 `none`，不能靠缺字段绕过模板证据门。

先写清：目标类目、目标受众、地域/语言、时间窗、可用人物/场景/道具、是否出镜、单篇制作成本、版权与安全禁区。数字来自用户窗口、当前研究口径或预注册，不把固定天数、互动量或账号规模当平台标准。

## 2. 模板不是一种东西

每个候选可同时标多个 `template_type`，但必须指出主机制：

| 类型 | 可重复的最小单元 | 观察重点 |
|---|---|---|
| `ranking_scale` | 统一等级轴与评价槽位 | 评价对象、排序语言、争议接口 |
| `shooting_action` | 人物/物体动作与机位关系 | 景别、动作节点、所需场景 |
| `transition_grammar` | 前后镜头的衔接规则 | 遮挡、甩镜、卡点、动作匹配 |
| `editing_grammar` | 节奏、字幕、画中画、音画关系 | cut 点、节拍、信息释放 |
| `meme_remix` | 固定母题加新的语境解释 | 母题识别、改写空间、版权 |
| `challenge` | 明确规则、限制和完成条件 | 门槛、失败笑点、交作业路径 |
| `interactive_prompt` | 让观众选择、补充或站队 | 评论槽位、答案延迟、复刻意愿 |
| `story_carrier` | 固定叙事顺序与角色槽位 | 开场冲突、转折、结尾兑现 |
| `visual_carrier` | 固定的信息承载形式 | 字卡、截图批注、九宫格、实拍证据 |

同一种贴纸、字体或 BGM 不足以证明同一模板；相同底层语法换了类目、人物和画面，才可能属于同一 `family_id`。

## 3. 发现词树

按“命名趋势发现 + 无元词结构发现 → 载体收窄 → 精确追链 → 反证”四层搜索。每条查询写入 `query-log.csv`；趋势候选的 `discovery_queries` 另保存 query ID、父 query、轮次和派生来源。

### 3.1 高召回元词

这些词用于找到平台用户正在怎样命名趋势，不用于直接下结论：

```text
最近爆火 拍照模板 / 视频模板 / 拍摄灵感
全网都在拍 / 全网都在玩 / 最近被这个挑战刷屏
跟风拍一个 / 跟拍 / 交作业 / 同款怎么拍
模板教程 / 原图 / 空白模板 / 一键生成 / 保姆级教程
最近很火的 转场 / 姿势 / 梗图 / 互动图 / 挑战
```

同时搜口语变体、错别字、英文名和缩写。元词结果往往混有教程营销、旧模板合集和跨平台搬运，必须继续追原帖。

### 3.2 载体与动作词

从当前类目的真实生产条件选词，不机械全搜：

```text
转场 / 变装 / 卡点 / 遮镜 / 甩镜 / 动作匹配
拍照姿势 / 怼脸照 / 合照 / fit check / 运镜
双重曝光 / 画中画 / 截图批注 / 九宫格 / 对比图
梗图 / meme / 互动图 / 排名 / 测评 / 从夯到拉
随机挑战 / 憋笑挑战 / 限时挑战 / 模仿挑战
```

把 `{类目/人群/场景}` 与载体词交叉。例如不是只搜“转场”，而是搜“宿舍转场”“情侣转场”“产品前后转场”，检验是否跨场景迁移。

### 3.3 结果页派生词

每发现一个候选，从以下位置继续长词：

- 标题中反复出现的完整短语与句式骨架；
- 话题、模板名、BGM/音频名、滤镜名、挑战名；
- 评论里的“求模板”“怎么拍”“原图”“交作业”及用户自发别名；
- 创作者正文里对来源、原版、教程或灵感的归因；
- 搜索框联想词与相关搜索，但联想只作为新 query，不当热度证据。

### 3.4 反证词

对每个候选主动补：

```text
{精确模板名} 翻车 / 尴尬 / 过时 / 拍烂了
{精确模板名} 教程 / 原图 / 空白模板 / 生成器
{精确模板名} 最新 / 低赞 / 普通账号 / 不同类目
{精确模板名} 原版 / 来源 / 抄袭 / 版权
```

“教程、空白模板、生成器大量出现”常提示模板进入成熟期，不等于仍在上升；“拍烂了”是线索，不是结论，仍要看当前样本。

### 3.5 无元词 feed 结构发现

只搜“爆火/模板/全网都在拍”会偏向已经被命名、SEO 化或进入成熟期的趋势。每次雷达同时保留一条不含这些元词的发现轨：

1. 在用户给定的当前时间窗内，从目标类目 feed、最新结果和相邻场景抽取近期样本。
2. 先盲看 hook、镜头动作、cut 点、字幕/音画关系和参与规则，不按标题里的“模板”二字入选。
3. 把结构相近的帖子暂归 `unnamed_structure_cluster`，再反查是否跨独立账号、类目或 BGM 复现。
4. 若之后搜到用户命名、教程或话题，再合并到同一 family；搜不到名称也可保持未命名 candidate。

`discovery_lanes` 记录 `named_trend | unnamed_structure_cluster`。评论参与只是可选扩散证据；低评论但独立创作者快速重复的视觉/剪辑语法仍可进入 replicated，不能因不适合“交作业”而被漏掉。

## 4. 精确短语追链与二创链

宽搜只负责发现名字；判断模板必须做精确短语追链：

1. 复制候选标题中最有辨识度、可复用的短语骨架，站内精确搜索。
2. 去掉类目词再搜，判断是否跨类目；替换一个槽位再搜，判断是否存在结构性变体。
3. 搜别名、错别字、话题名、BGM/音频名和评论里出现的民间叫法。
4. 打开最早可确认的来源线索、当前新作、明显高公开代理样本和普通/低样本。
5. 用公开搜索引擎或其他公开平台搜精确短语，只用于寻找来源时间、跨平台扩散或别名；当前小红书模板状态仍要回到站内近期样本判断。
6. 记录每个二创与母题的 `remix_relation`：`direct_remake | slot_substitution | category_transfer | parody | tutorial | generator | unrelated_same_phrase`；同时写 `creative_family_id`、`lineage_cluster_id` 和 `supply_origin=spontaneous|brand_campaign|platform_program|paid_distribution|mixed|unknown`。

这一步的输出不是“搜到很多篇”，而是一条可审计二创链：

```text
候选短语 → 原始/早期线索 → 独立账号复刻 → 类目变体
         → 教程/空白模板 → 当前普通或失败样本
```

多个结果来自同一搬运、同一 MCN 素材或同一账号时，不能冒充独立复刻。按 `note_id/URL/creative_family/account_id` 去重。

## 5. 打开原帖并完成结构采样

搜索卡片只能做候选。每个准备晋级的模板至少要打开能承担这些角色的样本：

- `seed`：可确认结构、早期来源或代表性强的母题；
- `support`：不同账号使用同一底层语法的近期复刻；
- `counterexample`：同模板普通/低表现、翻车或不适配样本；
- `boundary`：看起来相似但底层语法、任务或载体不同的样本。

逐帖记录：

1. 标题、作者、日期、URL、查询、入口、排序、可见位置和采集时间。
2. 页面实际显示的公开互动；缺失写 unknown，不拆合并值、不补 0。
3. 同账号近期可比基线与 `performance_tier`；无法比较时写 unknown。
4. 前 3 秒/首屏承诺、镜头顺序、动作节点、cut 点、字幕/BGM 关系和结尾。
5. 模板之外的主题、人物、账号体量、发布时间、投放/事件等混杂。
6. 评论参与证据：求模板、问拍法、交作业、视频回复、贴图复刻、主动报结果。
7. 质疑、审美疲劳、版权/来源争议和安全风险。

评论只证明“部分可见用户怎样参与”，不能估算全体比例。创作者置顶话术、重复口号和推广评论与普通用户自发参与分开编码。

## 6. 归一成模板家族

先写结构，后起名字。每个模板卡必须能回答：

```text
hook_grammar：观众第一眼看见什么未完成承诺？
shot_grammar：人物/物体/机位按什么顺序动作？
edit_grammar：在哪些节点切、遮挡、卡点或释放答案？
participation_loop：观众怎样复刻、站队、补充或交作业？
slot_map：哪些槽位必须固定，哪些槽位允许替换？
material_gate：复刻需要哪些真实人物、地点、道具、截图或证据？
```

归一规则：

- 相同机制、不同类目：同一 `family_id`，不同 `template_id` 或 category variant。
- 相同表面元素、不同参与规则：不同家族。
- 只是换标题、配色、字体：不是新模板。
- 只是复制原作者人物、台词、画面：不是合法改编方案。
- 教程、空白模板、生成器是家族的扩散证据，不与成品复刻混成同一表现样本。

`canonical_name` 用中性结构名；热梗原名和用户别名放 `aliases`。这样热梗名称消失后，底层载体仍可检索。

## 7. 同模板高低对照

模板判断要把“结构是否流行”和“某篇是否表现高”拆开。

优先建立同模板高低对照：

1. 同类目、相近载体、相近发布时间的不同账号复刻。
2. 同账号使用与未使用该模板的近期可比内容。
3. 同模板的早期、当前和普通/低表现版本。
4. 同底层语法但不同主题的迁移版本。

对照时把账号体量、主题热度、人物吸引力、事件、投放、首发优势、制作质量和发布时间列为替代解释。若真正不同的只是人物、题材或素材质量，模板只能标 `series_constant/task_fit`，不能晋级为效果机制。

## 8. 生命周期判定

不要把“复刻证据成熟度”和“时间阶段”揉成一个枚举。分别记录：

| `replication_status` | 允许的判断 |
|---|---|
| `query_candidate` | 只得到关键词、联想、摘要或用户线索，尚未打开并核验原帖 |
| `observed` | 已完整看见至少一个样本并提取结构，但没有独立复刻证据 |
| `replicated` | 多个独立账号与 lineage 重复同一底层语法，搬运与同源素材已去重 |

`lifecycle_phase` 再使用以下时序状态；状态由可比窗口和事实条件推导，不用通用点赞阈值：

| 状态 | 允许的判断 |
|---|---|
| `unknown` | 没有两个可比时间切片或上一版快照，不能判断上升/衰退 |
| `rising` | 至少两个可比窗口或上一版快照显示独立复刻仍在新增；评论参与可加强但不是必需条件 |
| `mature` | 至少两个可比窗口仍有独立复刻，且跨类目复刻、教程、空白模板或生成器已经普及，当前结果开始明显分化 |
| `fatigued` | 可比旧/新窗口显示新增复刻回落或当前普通/低样本占主导，且没有新语义增量 |
| `evergreen_carrier` | 多个可比窗口持续存在但没有短期集中上升，更像稳定内容载体而非当期热梗 |

生命周期理由必须写事实链：`window_comparisons`、样本窗口、独立账号/lineage、最近复刻、类目跨度、教程/生成器、可选评论参与和反例。每个 comparison 固定为 `start / end / support_sample_ids[] / independent_derivatives`：sample 必须真实发布在该窗口内，每条 support 恰好进入一个不重叠窗口，独立数由 account × lineage × post identity 重新计算。少于两个有效窗口时一律 `unknown`，不能从“都很新”、空对象或教程关键词直接写 rising/mature。

旧卡每次调用前复查 `last_refreshed_at` 和当前窗口；复刻状态与时序阶段可以分别升级或降级，不要静默覆盖历史版本。

## 9. 拍不拍的决策门

不用伪精确总分。逐项过门后给一个动作：

| 动作 | 机器值 | 适用条件 |
|---|---|---|
| 拍 | `shoot` | `replicated × rising/mature/evergreen_carrier`，账号受众与素材天然匹配，能加入新解释，权利与安全已通过 |
| 改后拍 | `adapt` | 同样先过 replicated 与合法时序门；原版题材/人物/成本不适配，但能保留语法并替换槽位 |
| 观察 | `observe` | 仍是 query_candidate/observed，近期复制或反例不足，先设下一轮精确查询 |
| 不追 | `skip` | 已疲劳、只有单篇高赞、主要依赖不可得素材、复制风险高，或与账号目标/规则冲突 |

按顺序判断：

1. **证据完整**：原帖、复刻、评论、低表现对照和日期是否可核验。
2. **时序**：新作是否仍在出现，还是旧帖长期占据热门结果。
3. **迁移**：去掉原作者后，底层语法还能否服务本账号的一个具体任务。
4. **素材**：真实人物、场景、道具、案例或截图是否已经拥有。
5. **增量**：本稿能否增加新解释，而非只换名词。
6. **权利与安全**：BGM、原图、人物、商标、未成年人、危险动作和敏感内容是否可控。
7. **衰减风险**：制作周期是否可能晚于模板有效期；无法判断时选择观察或 evergreen 改编。

## 10. 落库合同

运行目录新增：

```text
trend-template-samples.csv       # 每个打开并编码的帖子一行
trend-template-candidates.jsonl  # 每个模板家族/变体一条可版本化候选卡
```

分别复制：

- `assets/trend-template-samples-template.csv`
- `assets/trend-template-candidates-template.jsonl`

落库顺序：

1. 先写 `query-log.csv`、`source-log.csv` 和 `posts.csv`。
2. 再写逐帖 `trend-template-samples.csv`，保留 seed/support/counterexample/boundary 角色。
3. 去重并形成模板家族后，写本 run 的 current candidate snapshot；每行有不可变 `candidate_record_id + record_sha256`。同一 `template_id` 刷新时递增 `candidate_version` 并填写上一版 `supersedes_candidate_record_id`。长期库追加保存历史版本，运行文件每个 template 只保留本次使用的一张 current snapshot。
4. 卡片的 sample IDs 必须能回到逐帖行；逐帖行的 `post_id/note_id/canonical URL/account_id/published_at` 必须与 `posts.csv` 同一记录一致，post/note/canonical URL 不能跨 seed/support/counterexample/boundary 重复占位。post/source/query IDs 必须可解析。
5. 公开第三方内容只保存必要结构观察、短摘要、链接和哈希；不把原视频、完整图片或全文提交到公开仓库。

模板卡至少保留：发现词、来源/支持/反例、独立账号数、类目跨度、时间窗、四段语法、真实素材门、生命周期与理由、证据边界、决策、改编约束和刷新时间。

## 11. 生成时调用

不能因为库里有一张热梗卡，就直接套文案。生成时按以下顺序：

1. 用户要求“近期/现在”时，先用同口径窗口刷新候选卡；旧卡不能直接当当前趋势。
2. 读取本 run 每个 `template_id` 唯一的 current snapshot，并确认它是长期库中哈希有效的最高 `candidate_version`；再检索 `replication_status=replicated + lifecycle_phase=rising|mature|evergreen_carrier + decision=shoot|adapt`。素材门、类目、carrier、primary job、traffic stage、权利和安全约束必须 exact 相符。
3. 绑定的是 `hook_grammar + shot_grammar + edit_grammar + participation_loop`，不是原作者的标题、人物、画面、音频或独特台词。
4. 把 `required_material_codes` 映射为 JSON string array，数组中的每个 ID 必须能回到本 run 的 Source / Claim / Post / Authorization / authorized material。任一必需素材缺失或 ID 不可解析，返回 `needs_materials` 或改选载体。
5. 输出本次 slot map：固定什么、替换什么、新增什么解释、故意偏离什么。
6. 将模板卡的 support/counterexample IDs 写入创作简报；公开代理仍是 `public_proxy_association` 上限。
7. 封面、正文和分镜继续通过 traffic mechanism、style binding、真实性、授权、版权、商业与合规硬门；模板卡不替代这些门。

一个可复核的调用记录：

```text
candidate_record_id / template_id / family_id / candidate_version
→ replication_status × lifecycle_phase + last_refreshed_at
→ selected decision + reason
→ source/support/counterexample sample IDs
→ fixed slots / replaced slots / new semantic contribution
→ material code → [actual local evidence/material IDs]
→ title / cover / shot list / edit points / comment action
→ failure condition + next refresh query
```

`mature` 模板通常优先“改后拍”，通过人群、场景或观点增加语义；`fatigued` 默认不直接生成，除非把它当可识别母题进行明确反转，且新解释本身有证据和素材。

## 12. 停止条件与常见误判

正常停止沿用 [research-method.md](research-method.md) 的覆盖、饱和和最多四轮查询树。趋势任务还要满足：候选已归家族、至少存在一个当前样本角色和一个反证角色、生命周期理由可追溯。

以下情况必须降级为 `observe/query_candidate`，不能输出“近期爆火模板”：

- 只有搜索摘要、个性化首页或单篇高赞；
- 没打开原帖，无法确认实际镜头/页面结构；
- 结果都来自同一账号、同源搬运或同一活动供给；
- 只有旧热门，没有当前复刻；
- 只有教程营销，没有真实成品与评论参与证据；
- 缺普通/低表现对照，无法判断是模板还是题材/人物在工作；
- 日期含混，无法纳入本次时间窗；
- 遇到登录墙、验证码、人机验证或风控。

不要把“搜索结果很多”“教程很多”“评论都在求模板”单独写成流量因果。它们分别支持供给、成熟度或参与解释，不能替代曝光数据。

## 13. 输出合同

先给决策表，再给证据，不堆模板名：

| 模板 | 状态 | 动作 | 为什么现在 | 本账号怎么改 | 必需素材 | 主要风险 | 证据/反例 |
|---|---|---|---|---|---|---|---|

随后附：

1. 本轮发现词树和精确追链 query。
2. 去重后的样本角色与时间窗。
3. 每个模板的四段语法和 slot map。
4. 生命周期理由与替代解释。
5. 落库路径、版本和下一次刷新条件。

所有表述限定于本次可见样本、时间窗、入口和访问条件。没有一方曝光数据时，交付物只能说“近期重复供给”“公开代理异常”“评论参与信号”或“候选模板”，不能说“拿到流量”“爆款公式”或“已验证”。

## 14. 自检

- [ ] 不是从一篇高赞直接抽模板。
- [ ] 已从元词进入精确短语追链，不只搜“爆款模板”。
- [ ] 已区分母题、复刻、教程、生成器、搬运与无关同名。
- [ ] 已打开原帖，读取镜头/页面、正文、日期、公开指标和评论。
- [ ] 已做独立账号去重、模板家族归一和同模板高低对照。
- [ ] 生命周期理由包含近期样本、扩散、参与和反例，不靠固定阈值。
- [ ] 决策为拍/改后拍/观察/不追之一，且通过真实素材与权利门。
- [ ] 生成只复用语法，不复制原句、人物、画面、音频或独特资产。
- [ ] 两个模板文件、query/source/post 引用和限制已经落盘。
- [ ] `public_proxy ≠ traffic`，没有把公开互动或搜索位置写成流量因果。
