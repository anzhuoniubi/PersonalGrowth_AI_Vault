# 获客、评论与跨平台承接执行规范

本文件供执行 `redbook-writing` 的代理制定获客、评论、CTA、成交承接和跨平台分发方案，解决“能否做、在哪做、如何测”，不提供规避审核、暗号导流或多账号截流方法。

导航：[方向与规则](#2-四种渠道方向) · [资格门](#4-sku--offer--platform--account--surface--creative-资格门) · [评论边界](#5-评论区三色门) · [CTA](#7-cta-决策) · [归因实验](#10-跨平台归因) · [交付](#13-最终决策表与交付格式)

## 1. 执行原则

按以下优先级决策：

1. 法律法规与具体商品资质。
2. 平台当前规则与后台实际配置。
3. 账号主体、行业、SKU、offer 和发布 surface 的资格。
4. 用户安全、知情和隐私。
5. 内容质量与渠道原生性。
6. 增长效率。

采用以下默认值：

| 事项 | 默认值 | 何时可以改变 |
|---|---|---|
| 小红书手工外跳 | `blocked` | 当前官方产品明确开放，且具体主体、行业、SKU、物料均审核通过 |
| 评论区商业获客 | `blocked` | 不设例外；改走官方商业组件或自有内容 |
| 资格信息缺失 | `unknown → blocked` | 补齐可复核证据后重新判定 |
| 跨平台精确归因 | `unavailable` | 存在官方跨域归因或经同意的确定性链路 |
| 真实投稿公开 | `blocked` | 完成年龄、授权、脱敏、安全和撤回检查 |
| 虚构内容冒充真人真事 | `blocked` | 不设例外；必须标注“虚构演绎”或“综合改编” |

不要把点赞、收藏、粉丝、评论数和单篇曝光直接等同于生意结果；不要把“行业里有人在做”、未处罚、暂时发布成功或竞品仍在线当作允许证据。

## 2. 四种渠道方向

每个渠道动作必须且只能选择一个 `direction`。

| direction | 定义 | 合法目标 | 默认承接 |
|---|---|---|---|
| `external_to_xhs` | 站外需求或受众进入小红书品牌内容 | 品牌搜索、主页访问、关注、系列阅读 | 小红书主页或指定笔记 |
| `xhs_to_native_conversion` | 用户留在小红书内完成下一步 | 店铺、商品卡、合规私信、留资页、小程序、直播 | 平台原生组件 |
| `xhs_to_approved_external` | 通过小红书官方批准的产品跳转站外 | 获批第三方购买或服务承接 | 官方外跳组件 |
| `owned_retention` | 已自愿订阅或交易的用户进入自有/许可渠道 | 留存、服务、复购、召回 | 邮件、会员、自有站或合规私域 |

不得把以下动作错误归入 `external_to_xhs`：

- 在小红书评论里让人“看主页”。
- 用私信索要或发送站外联系方式。
- 用谐音、拆字、表情、二维码变体或暗号提示站外账号。
- 在竞品笔记下伪装用户推荐自己。

这些动作属于违规或高风险外引，不是合规跨平台分发。

## 3. 运行时规则复核

平台规则、类目、组件和后台资格会变化。涉及商业、成人、医疗健康、外跳、私信、评论或广告时，必须运行时复核。

### 3.1 证据优先级

| 等级 | 证据 | 可用于什么 |
|---|---|---|
| A | 法律法规、平台当前官方规则、登录后台资格结果、人工客服书面答复 | 资格与禁止项决策 |
| B | 平台官方产品页、官方培训材料、官方案例 | 能力发现与方案候选 |
| C | 可靠媒体采访、学术研究、奖项案例 | 假设与风险提示 |
| D | 运营帖、课程、工具商、匿名口述 | 只作待验证线索 |

资格判定至少需要一条 A 级证据。

B、C、D 级证据不能把 `blocked` 改成 `allowed`。

### 3.2 复核记录

每次复核记录：platform、surface、精确 `sku_id`、`offer_id`、账号类型、来源URL、章节或后台路径、检查日期、registry 状态、方案判定、附带条件、证据路径和复核人。

截图只能辅助原始规则，不得用脱离上下文的截图替代可访问来源。

### 3.3 运行时必查入口

以下入口会变化，只作为复核起点：

- 小红书专业号规则：`https://ad.xiaohongshu.com/next_help/docs/195c5fe505c71b4b0335a2fe0d61d8e0`
- 小红书蒲公英内容审核：`https://pgy.xiaohongshu.com/help/detail?id=6495c527d1eedeeb48fb18b1f875650e&userType=4`
- 小红书聚光帮助中心：`https://ad.xiaohongshu.com/next_help/home`
- 国家市场监管总局《互联网广告管理办法》：`https://www.samr.gov.cn/zw/zfxxgk/fdzdgknr/fgs/art/2023/art_d93a579afd45413e8576e4623fab348f.html`
- 《个人信息保护法》：`https://www.npc.gov.cn/npc/c2/c30834/202108/t20210820_313088.html`

## 4. SKU / offer × platform × account × surface × creative 资格门

不要按“成人用品”“女性内容”这种大类直接判断。

对每个具体组合建立资格元组：

```text
eligibility_tuple = exact SKU or service
                  × exact offer
                  × exact platform
                  × exact account/entity
                  × exact surface
                  × exact creative ID and SHA-256
```

### 4.1 字段定义

| 维度 | 必须明确的内容 | 常见错误 |
|---|---|---|
| SKU | 品牌、型号、材质、用途、是否医疗器械、资质 | 用“情趣用品”代替具体商品 |
| offer | 售卖、试用、咨询、订阅、留资、联名、佣金、赠品 | 只写“转化” |
| platform | 小红书、知乎、B站、抖音、微信、自有站 | 假定各平台规则一致 |
| account | 个人号、专业号、企业号、店铺主体、广告主 | 用个人号承担企业销售 |
| surface | 笔记、评论、私信、主页、直播、店铺、广告、落地页 | 只确认账号，不确认发布面 |
| creative | 标题、正文、图片、视频、产品露出、CTA | 只确认商品，不审物料 |

### 4.2 资格状态

| registry 状态 | 定义 | 方案动作 |
|---|---|---|
| `confirmed` / `approved` | 精确 platform × account × surface × source_asset_id × SHA-256 当前有效；有本次运行采集的官方工单/审批记录及同元组 `sku_eligibility`/`offer_eligibility` 主张 | 全部 registry 行通过且素材字节版本匹配后，方案才可 `allowed` / `conditional` |
| `needs_*` / `insufficient_*` | 待平台确认、SKU输入或官方证据不足 | `unknown → blocked` |
| `expired` / `rejected` / `blocked` | 证据失效、拒绝或明确禁止 | `blocked` |

### 4.3 资格决策表

| 检查 | 是 | 否 / 不知道 |
|---|---|---|
| 商品或服务依法可经营吗 | 继续 | `blocked` |
| 具体主体拥有所需资质吗 | 继续 | `blocked` |
| 平台当前允许该精确SKU吗 | 继续 | `unknown → blocked` |
| 当前账号类型可经营吗 | 继续 | 更换合规主体或停止 |
| 当前surface允许该offer吗 | 继续 | 选择获批surface或停止 |
| 素材通过成人、低俗、医疗、广告标识检查吗 | 继续 | 修改内容；不可用暗语替代 |
| CTA由当前平台明确支持吗 | 继续 | `blocked` |

### 4.4 成人敏感品额外门

至少检查：平台禁入子类；后台实际分类；拟态器官、敏感部位、使用动作或性联想；催情、壮阳、治疗等功效声称；未成年人触达；广告审查、医疗器械备案和经营许可。

规则文本出现术语冲突时，不自行解释有利含义；把状态设为 `unknown → blocked`，用具体SKU向平台复核。

`xhs_to_approved_external` 只要渠道不是 blocked，就必须引用当前、精确匹配的 offer eligibility；若 offer 属于商品售卖/试用，还必须同时引用 SKU eligibility。两者都要绑定同一平台、账号、`approved_external_destination`、creative ID 与 SHA-256，并有具体目的地。即使 `permitted_cta=none` 也不能绕过，因为激活外跳路径本身就是商业能力判断。

## 5. 评论区三色门

评论的首要目标是补充公共讨论，不是获客。

### 5.1 绿门：可生成候选的公共贡献

同时满足以下条件才是绿色：

- 与原帖问题直接相关。
- 提供事实、经验边界、公开来源或具体帮助。
- 不提自己商品、价格、优惠、咨询、店铺或联系方式。
- 不要求看主页、看置顶、私信、搜索账号或转到别处。
- 不假装购买者、患者、路人或与品牌无关的人。
- 不复制模板，不批量发，不使用自动化。
- 不贬低原作者或竞品。
- 即使没有任何主页访问，这条评论仍有独立价值。

绿门评论只可生成一条候选，交由发布者逐字审核和本人手动决定；可以用于社区参与和用户研究，不能承诺流量结果，也不由 Skill 代为发布。

### 5.2 黄门：必须停下复核

命中任一项即为黄色：

- 原帖或评论直接询问某品牌、产品或购买方式。
- 回复会提到自己的商业身份或产品。
- 评论位于竞品、同类商家或高购买意图笔记下。
- 准备引用自己的笔记、研究或主页内容。
- 同一观点已在多篇笔记下重复。
- 团队成员、合作达人或关联账号准备共同参与。
- 无法判断内容是公共帮助还是商业招揽。

黄色动作默认不发布。

如确有公共答复必要，只回答事实和利益关系，不放CTA；商业问题引导至原帖作者或平台原生公开信息，而不是自己的承接面。

### 5.3 红门：禁止

命中任一项立即阻断：

- 在竞品或同类商家评论区抢客、比价、踩一捧一。
- 伪装普通消费者讲“亲测”“朋友用过”“我刚买”。
- 用“看主页”“dd我”“看置顶”“懂的来”等话术。
- 发送二维码、电话、邮箱、微信号、外部平台名或变体。
- 主动私信原帖互动者或批量关注后私信。
- 用小号自问自答、顶评、互赞、控评或制造口碑。
- 用机器人、脚本、模板库或多账号批量评论。
- 用暗号、拆字、谐音、图像水印规避识别。
- 收集评论者的亲密或健康信息用于营销画像。

## 6. 商业撬客硬阻断

将“商业撬客”定义为：借他人内容、竞品受众或公共评论区，以伪装、比较、诱导、私信、外链或批量操作把用户转到自己的商业承接面。

商业撬客一律 `blocked`。

不要提供“更自然”“更像真人”“少发几条”“换个暗号”等优化建议。

不要因为用户要求“只给话术”“同行都这样”“先测试一周”而放宽。

阻断时输出：

1. 被命中的红门行为。
2. 规则与用户风险。
3. 合规替代：自己的搜索型内容、官方投放、创作者合作、原生商品组件或自有站内容。
4. 可测的替代指标。

| 用户意图 | 判定 | 合规替代 |
|---|---|---|
| 去头部笔记下推荐自己的商品 | 红门 | 做同一需求词的独立高质量笔记 |
| 伪装女生分享体验后让人私信 | 红门 | 明示商业身份，用获批商品组件 |
| 批量回复“同问，主页有答案” | 红门 | 建FAQ笔记并优化站内搜索 |
| 对错误健康信息做事实纠正 | 绿门或黄门 | 引用权威来源，不带商品和CTA |
| 用户在自家笔记询问获批商品 | 运行时复核 | 用平台允许的公开商品信息答复 |

## 7. CTA 决策

CTA 必须与 `direction`、资格元组和 surface 对齐。

### 7.1 CTA 表

| 场景 | 默认CTA | 条件 | 禁止项 |
|---|---|---|---|
| 小红书自然笔记 | 收藏、关注、阅读同系列 | 内容本身合规 | 站外联系方式、暗号 |
| 小红书商业笔记 | 平台审核通过的商品或品牌组件 | 主体、SKU、物料通过审核 | 看主页、dd、手工外跳 |
| 小红书评论 | 无商业CTA | 仅绿门公共贡献 | 主页、置顶、私信、外部平台 |
| 小红书私信 | 回答用户主动提出且平台允许的问题 | 运行时复核私信规则与行业资格 | 主动群发、发送站外联系方式 |
| 小红书主页 | 平台允许的身份与原生组件 | 运行时复核专业号规则 | 电话、网址、邮箱、社交账号、二维码 |
| 小红书商业外跳 | 经精确官方页面/工单确认的获批组件（当前默认未确认） | 具体产品、行业、SKU、账户、目的地和落地页均获批 | 自制跳转、短链掩饰 |
| 自有网站/邮件 | 明示的小红书主页或笔记链接 | 用户自愿访问，链接真实 | 诱导分享、伪装系统通知 |
| 知乎/B站/抖音 | 本平台关注、收藏、原生商品组件 | 按各平台当前规则 | 未经许可的跨平台营销导流 |
| 线下活动 | 公开品牌名、活动信息 | 年龄、拍摄、场地和隐私合规 | 强制打卡、奖励发帖、采集敏感信息 |

### 7.2 外跳默认阻断

满足以下全部条件才可把 `xhs_to_approved_external` 从 `blocked` 改为 `conditional` 或 `allowed`：

1. 使用小红书当前官方外跳产品，而不是手工话术。
2. 广告账户和主体有资格。
3. 精确SKU或服务有资格。
4. 前端物料与后链路页面均通过审核。
5. 广告标识、隐私告知、价格和资质信息完整。
6. 有可审计的批准记录和检查日期。

任一条件缺失时保持 `blocked`。

## 8. 渠道角色与承接

不要把每个平台都做成小红书内容的复制品。

| 渠道 | 主要角色 | 原生载体 | 推荐direction | 承接 |
|---|---|---|---|---|
| 小红书 | 需求发现、搜索、信任、原生成交 | 图文、短视频、直播、店铺 | `xhs_to_native_conversion` | 获批原生组件 |
| 知乎 | 长尾问题与搜索解释 | 问答、文章 | `external_to_xhs` | 同名品牌记忆或明示笔记链接 |
| B站/播客 | 深度解释与信任 | 访谈、讲解、案例拆解 | `external_to_xhs` | 同名品牌记忆或明示笔记链接 |
| 抖音/视频号 | 广覆盖与命题测试 | 单点短视频、直播 | `external_to_xhs` | 同名品牌记忆；具体外链须获批 |
| 微信/邮件 | 自愿订阅后的留存、售后、复购 | 长文、通知、服务内容 | `owned_retention` | 合法订阅、服务与原生交易 |
| 自有网站 | 规则说明、内容归档、可控测量 | 专题页、FAQ、产品页 | `external_to_xhs` 或 `owned_retention`，按动作拆行 | 明示链接与一方分析 |
| 线下活动 | 体验、信任、自愿UGC | 沙龙、展览、工作坊 | `external_to_xhs` | 品牌搜索和自愿分享 |

外部平台提及小红书前，运行时复核该平台的商业推广和外链规则。

无明确允许证据时，保留一致、公开的品牌名称，不写跨平台行动指令。

## 9. 指标可得性

分开记录 `metric_availability` 和 `attribution_level`，再制定目标。

| 字段和值 | 定义 | 决策用途 |
|---|---|---|
| availability=`available` | 数据源、权限和事件定义已确认 | 可定目标 |
| availability=`needs_backend_check` | 取决于账户后台或组件权限 | 核验前不定目标 |
| availability=`unavailable` | 不记录或无法可靠分离 | 不得编造或反推 |
| attribution=`platform_native` | 同一平台原生组件与后台事件 | 只用于该平台定义的事件与窗口 |
| attribution=`user_level_with_consent` | 有合法同意的确定性用户级链路 | 记录标识、用途、退出与保留边界 |
| attribution=`modeled` | 有口径透明、可审计的归因模型 | 报模型假设与不确定性 |
| attribution=`directional` / `unavailable` | 只能观察聚合关联，或没有可解释信号 | 只报方向，或不归因 |

### 9.1 各方向指标

| direction | 主要指标 | 诊断指标 | 常见不可得指标 |
|---|---|---|---|
| `external_to_xhs` | 品牌搜索增量、主页访问增量、新关注增量 | 外站完读、自有链接点击、系列词搜索 | 用户级“看了知乎后关注小红书” |
| `xhs_to_native_conversion` | 商品点击、加购、支付、退款、复购或有效留资 | 笔记阅读、主页访问、私信发起 | 未接入组件的站外订单 |
| `xhs_to_approved_external` | 官方归因点击、获批落地页转化、订单 | 到达率、页面退出、退款 | 私下发送链接后的可靠归因 |
| `owned_retention` | 经同意订阅、复购、留存或服务完成 | 打开、阅读、退订、退款 | 未经同意的跨平台身份拼接 |

评论区不设“引流转化率”KPI。

绿门评论可以记录回复质量、被采纳问题和新选题，不把主页访问归因给单条评论，除非平台提供明确数据。

## 10. 跨平台归因

### 10.1 何时只能写 directional

命中任一项，跨平台结论只能设为 `attribution_level=directional`：

- 用户需要手动搜索同名品牌。
- 没有官方跨域归因产品。
- 没有经同意的确定性标识连接两端事件。
- 同期有多个渠道、达人、投放或热点曝光。
- 只能观察总粉丝、品牌搜索或总订单变化。
- 外部平台不提供点击，或小红书不提供来源。
- 样本小到单个大事件即可改变结果。

此时可以说：

> 活动期小红书品牌搜索和主页访问同步上升，与外部内容发布时间一致；由于缺少用户级或官方跨域归因，只能视为方向性关联。

不得说：

> 这篇B站视频给小红书带来了312个精准粉丝。

### 10.2 提升可信度

证据强度由高到低：官方跨域或广告归因；随机化实验；地域、时间或受众留出组；分渠道错峰；稳定基线加预注册中断时间序列；描述性前后对比。

不要用暗号或隐蔽口令改善归因。

可使用公开、稳定的系列名称观察品牌词变化，但它不是用户级标识。

## 11. 实验预注册

发布或投放前完成预注册；数据出来后不得改主指标和成功线。

| 必填项 | 内容 |
|---|---|
| 身份 | 实验ID、可证伪假设、`direction`、资格元组ID |
| 设计 | 渠道、受众状态、干预、分析单位、基线和测试窗口 |
| 测量 | 唯一主指标、`metric_availability`、护栏、诊断指标、归因方法与 `attribution_level` |
| 判断 | 混杂因素、最小样本或时长、决策规则、停止条件 |

预注册检查：只设一个主指标；写明定义、数据源、时区、窗口、不可得指标和基于基线或成本的成功线；记录同期投放、合作、热点和平台异常；规定最短观察期与风险停止条件。合规告警优先于增长结果。

## 12. 敏感投稿与聊天记录隐私

亲密关系、性经历、健康、取向、身份、创伤和联系方式均按高敏感信息处理。

### 12.1 来源类型必须外显

| source_type | 定义 | 对外标识 |
|---|---|---|
| `verified_submission` | 投稿者授权且完成必要核验 | “经授权投稿，细节已脱敏” |
| `composite` | 多个案例合并，人物和细节重构 | “根据多份投稿综合改编” |
| `fictional` | 完全虚构 | “虚构演绎” |
| `public_case` | 来自公开可靠来源 | 标明来源，不冒充亲历 |

不得把 `composite` 或 `fictional` 标成“真实聊天记录”“真人真事”。

### 12.2 收集前

收集前必须确认投稿者年满18岁；说明目的、使用方式、公开范围和期限；把接收、公开、商业使用拆成独立同意；提供撤回、修改、删除渠道；最小化字段；明确禁止提交可识别第三方、未成年人或未经同意的私密影像。

### 12.3 脱敏

删除或重构姓名、头像、账号ID、联系方式、精确年龄与单位、地点与日期、行程与订单号、独特事件组合、EXIF与设备信息、二维码与水印、可识别第三方的语言和群聊特征，以及非必要医疗健康细节。

优先重绘聊天界面，不直接发布原始截图。

重绘不能改变核心事实，也不能制造原投稿不存在的商品体验。

### 12.4 存储与访问

- 原始投稿与内容稿分开存储。
- 只给必要角色最小权限。
- 记录访问、编辑、导出和删除。
- 设置明确删除日期；项目结束不无限期保留。
- 不把原始敏感投稿写入公开研究库、通用提示词或第三方工具。
- 使用外部模型或服务前确认数据处理和跨境要求。

### 12.5 硬阻断

投稿者可能未成年、第三方未同意且可识别、涉及胁迫偷拍泄露报复或现实安全风险、撤回未完成、必须依赖真实身份或露骨细节、夹带未披露营销，或把抓取的评论私信群聊改成“投稿”时，一律不得发布。

遇到现实伤害、暴力、胁迫或医疗紧急风险时，停止内容生产；不要以账号运营替代专业援助。

## 13. 最终决策表与交付格式

### 13.1 一页决策表

| 问题 | 结果 | 下一步 |
|---|---|---|
| direction 已唯一确定吗 | 否 | 先拆分渠道动作 |
| 资格元组完整吗 | 否 | 补齐SKU、offer、platform、account、surface、creative |
| 有当前 A 级资格证据吗 | 否 | `unknown → blocked`，运行时复核 |
| 涉及评论获客吗 | 是 | 进入三色门 |
| 命中商业撬客吗 | 是 | 硬阻断并给合规替代 |
| 涉及小红书外跳吗 | 是 | 默认 `blocked`；只检查官方批准组件 |
| 涉及成人敏感商品吗 | 是 | 执行成人额外门和SKU人工确认 |
| 涉及投稿或聊天记录吗 | 是 | 完成年龄、授权、脱敏、存储和撤回检查 |
| 跨平台无确定性链路吗 | 是 | 只报 directional |
| 实验已预注册吗 | 否 | 发布或投放前补齐 |
| 主要指标可直接获得吗 | 否 | 降级结论或更换可测设计 |

### 13.2 代理交付合同

| 输出组 | 必填字段 |
|---|---|
| 判定 | `verdict`、`direction`、`eligibility_tuple`、`runtime_checks` |
| 执行 | `channel_role`、`native_format`、允许与禁止 `cta` |
| 测量 | `measurement`、`preregistration` |
| 安全 | `privacy`；若阻断还要给合规 `alternative` |

## 停止条件

平台规则、后台资格或人工答复冲突；收到违规、限权、驳回或隐私投诉；SKU重分类、下架或缺资质；方案依赖伪装、暗号、多账号或自动化；精确跨平台转化无数据来源；投稿授权、第三方同意或年龄不清时，立即停止发布、评论、投放或外跳。

停止不是实验失败；它是合规和用户安全护栏正常生效。
