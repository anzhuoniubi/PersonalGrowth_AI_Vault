# 条件性经验、玄学审计与实验

经验资料的价值是提出假设、补用户语言和发现失败模式，不是替平台公布算法。公众号、知乎、工具商、MCN、创作者帖子和服务商案例都要保留分母、样本、入口、时期、后台字段与失败样本。

## 四问审计

遇到任何“流量经验”先问：

1. 分母是什么：曝光、点击、阅读、互动、搜索、主页、私信还是订单？
2. 样本是什么：几个账号、什么类目、粉丝/账号阶段、自然还是付费？
3. 观察入口是什么：推荐、搜索、榜单、蒲公英、广告还是店铺？
4. 失败样本在哪里：普通笔记、反例、时间外数据和竞争变化是否保留？

再补三项：资料日期、原始后台/链接是否可见、作者是否销售课程或工具。有利益关系不等于内容错误，但必须影响证据等级。

## 证据等级的经验用法

| 等级 | 条件 | Skill 动作 |
|---|---|---|
| B | 有清晰方法/样本、可回原始报告，或多账号后台且限制明确 | 可写“该样本观察到”，不可写平台因果 |
| C | 单账号后台、访谈、工具商案例、抽样不全 | 生成条件性假设，主动找反例 |
| D | 无原始链接、无分母、只给精确口诀/收入/涨粉 | 标 rumor/unknown；不进入默认策略 |

论文或报告整体等级不能自动传给其中每个二手数字。精确算法公式若只来自访谈整理，即使论文整体为 B，也可对该主张单独降为 D。

## 可以保留的条件化经验

### 1. 漏斗胜过单一互动分

跨来源资料与公开技术材料都更支持分层诊断：

```text
曝光 → 点击 → 消费深度 → 收藏/评论/分享
                         ↓
                  主页/关注 → 搜索/私信/组件 → 订单
```

生产动作：每篇只选一个首要任务与主指标；不能用点赞替代收藏、搜索、关注或订单。评论“在问什么”往往比评论总数更能说明需求，但仍不能代表完整人群。

### 2. 搜索意图与推荐场景不同

搜索稿通常更需要明确问题、条件、步骤、对比和可回看答案；推荐稿通常先让用户认出具体处境或张力。这个差异来自用户任务，不是两套固定模板。标题、封面、正文、话题和答案位置仍需语义一致。

### 3. 高互动不等于商业结果

历史跨品牌时间序列曾出现相近互动对应差异极大的搜索变化，甚至搜索下降。只能支持“换算不稳定”，不能提供新的固定换算。商业稿同步记录品牌词、主页、经审私信/组件、订单、退款和投诉；缺后链路时标 `proxy_only`。

### 4. 载体按信息任务选择

- 图文/轮播优先测试：步骤、清单、对比、地图、价格、资料索引、判断表。
- 视频优先测试：过程、演示、动作、语气、人物关系、人格和情绪叙事。

历史“视频更高互动”常来自扶持期、商业样本或特定类目，不能外推为 2026 全站偏好。用本账号的单位制作时间、合格消费、收藏/关注/转化验证。

### 5. 生命周期不统一

把内容分为：

- `hot`：跟随事件和竞争，观察短窗。
- `periodic`：季节/节日/阶段性需求，与相同周期比较。
- `evergreen_search`：具体长期问题，除近期快照外保留一个由历史尾部消费推导的较晚观测位。

部分历史样本第 7 天后仍有明显消费，只能反驳“全部七天失效”；不能反向断言所有内容都长尾。

### 6. 定位主要改善可理解性

一个稳定用户承诺和少量彼此连贯的内容支柱有助用户理解主页、决定是否关注，也便于研究同类样本。未找到可靠证据证明“垂直度”是一个可由创作者操作的固定账号权重。低粉单篇高表现存在，但不代表关注/转化会同步发生。

## 玄学黑名单与替代方案

| 流传说法 | 证据状态 | 不能做 | 可验证替代 |
|---|---|---|---|
| 每篇先给 200/500/1000 人，再按阈值升级 | D/未找到一手依据 | 写固定流量池、及格率 | 记录本账号曝光分布与冷启动阶段，不承诺阈值 |
| CES=点赞×1、收藏×1、评论×4、分享×4、关注×8 | D/互相冲突 | 用公式打分或优化 | 按点击、消费、互动和后链路分层看 |
| 收藏/评论固定比点赞权重高 | unknown | 给通用权重 | 按类目、入口和目标做准实验 |
| 连续发七天获得垂类标签 | unknown | 设计“养标签”SOP | 稳定承诺、内容支柱和主页可理解性 |
| 晚上8点/午间必然最好 | D | 叫全平台黄金时间 | 用受众与账号自身数据测试时段 |
| 发得越多涨得越快 | contradicted/混杂 | 规定每天 N 篇 | 比较内容中位数与单位生产时间回报 |
| 低流量就是限流 | 错误归因 | 从结果倒推处罚 | 查需求→点击→承接→收录/竞争→规则/站内信 |
| 违规统一限流7/15/30天 | unknown/规则混淆 | 给统一恢复倒计时 | 按具体规则对象、提示、模块和工单处理 |
| 蒲公英健康异常拖累主站 | 官方反证 | 归因主站低流量 | 分开诊断蒲公英权限与 App 主站 |
| 真粉连续点赞会伤号 | 官方反证 | 劝粉丝停止正常互动 | 只阻断互赞、利益诱导、买量和水军 |
| 标签越多受众越精准 | D | 堆无关热门标签 | 使用真实相关词并保持全文一致 |
| 新号必须养号、互赞互粉 | 无依据且有作弊风险 | 制作互动群/养号流程 | 正常使用、真实内容、保留自然基线 |
| 视频一定比图文更好 | 类目/时期冲突 | 默认全做视频 | 按任务与制作约束匹配测试 |
| “商业感”会自动限流 | 概念混淆 | 教伪素人/弱化披露 | 合规披露、证据与内容价值分开审查 |
| 换封面/颜色即可复制爆款 | D | 给稳定爆率 | 同题只改一变量、多组比较中位数 |
| 搜索占 65%/70% | 分母混淆 | 把月活搜索渗透写成单篇流量 | 只用本账号后台入口占比 |
| 评论前五固定置顶/单条引固定人数 | 旧/无后台 | 评论截流 SOP | 评论仅作帮助/研究，不设获客 KPI |
| 外站粉丝固定比例迁移小红书 | 无用户级闭环 | 给精确迁移率 | 系列名、错峰、基线与对照主题，标 directional |

用户坚持要确定数字时，不要只拒绝。给出：原说法、证据状态、作用域、缺的分母/原始来源、当前能测的替代指标。

### “标准 SOP”压力门

用户索要前 7 天、30 天、每天几篇、固定配比、统一字数、页数或 24/72 小时复盘表时，先逐项追问数字从哪里来；不要在“这不是平台规则”的免责声明后，继续编一套同样无依据的数字。

数字只允许来自四类输入：

1. 当前官方或原始研究明确给出的数字，并保留模块、时期和适用对象。
2. 本账号同口径基线或后台可审计字段。
3. 用户真实产能、预算、库存、审核时延等业务约束。
4. 预注册实验根据基线推导的样本量、窗口和停止条件。

缺少以上输入时，SOP 写成依赖关系：`完成证据门 → 生产一篇合格内容 → 等主指标达到可解释事件量 → 只改一个变量 → 更新基线`。不要把它们重新编号成 Day 1—Day 7：那仍是在制造虚假的时间确定性。不得把研究采样量迁移为发布量，也不得把资料库里的历史观测时点改写成所有笔记的固定复盘日。

## 账号自然内容只能做准实验

自然发布不是随机 A/B。使用以下最小合同：

```text
hypothesis_id:
primary_job:
baseline_population:
matched_topic/account_stage/time_context:
single_changed_variable:
primary_metric:
metric_definition + availability:
diagnostic_metrics:
confounders:
observation_window:
minimum_events (derived from baseline, not a universal number):
decision_rule:
result: win / loss / inconclusive:
```

### 设计步骤

1. 先收集账号自身基线，确认目标指标真实可取。
2. 匹配题材、入口、账号阶段和大致时间环境。
3. 一次只改变标题、封面、载体、结构、发布时间中的一个变量。
4. 至少重复多组，用中位数和分布，不用一对笔记下结论。
5. 记录热点、竞争、审核、投放、合作、发布时间和样本污染等混杂。
6. 事件不足或方向不稳定时写 `inconclusive`，不为了复盘产出强结论。

2–3 个标题/封面版本只是候选，不代表一次发布能识别标题和封面各自效果。若同时改两者，只能评价组合，不能拆因果。

## 置信度必须由条件推导

### 单字段

- `high`：原页面直接可见，带稳定链接与采集日期。
- `medium`：截图/转述可核但字段或日期有一项不完整。
- `low`：搜索片段、含混相对日期或无法打开详情。

### 跨内容模式

- `high`：至少 3 个独立账号、多条近期样本、至少 2 种查询路径，并保留反例。
- `medium`：至少 2 个独立账号，或同账号近期反复出现，但受众/原因仍含推断。
- `low`：单篇、日期/指标缺失、评论抽样少或人群含混。

“为什么爆”“评论区代表谁”通常最高 `medium`；没有后台或独立证据不得升级。

## 经验结论写法模板

```text
观察：在 [日期/入口/样本] 中看见 [字段或模式]。
计算：按 [统一口径] 得到 [中位数/异常倍数/比例]。
解释：可能与 [机制] 有关，但 [混杂/反例] 尚不能排除。
范围：只适用于 [账号阶段/类目/入口/时期]。
动作：把它作为 [选题/载体/时段] 假设，用 [主指标] 做准实验。
```

不要把“解释”那一行删掉不确定词后当标题结论。
