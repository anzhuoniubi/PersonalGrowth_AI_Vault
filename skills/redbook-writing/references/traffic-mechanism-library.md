# 小红书统一流量机制库 v1

> 快照：2026-07-18
>
> 机器资产：`assets/traffic-mechanisms-v1.json`
>
> 作用：把公开课程、品牌/服务商项目、生产研究、意外公开产物和站内高低公开代理样本，压成跨类目、可检索、可执行、可证伪的内容机制。
> 边界：**不承诺爆款**。没有具体账号的一方 `impressions/reach` 复现时，只能称生产机制、任务适配或待验证表现假设。

## 1. 这套库解决什么

它不回答“什么颜色、字体、便签或大字最容易爆”，而回答三个更接近生产的问题：

1. 这一篇首先要推动哪个流量环节？
2. 用户要完成什么任务，现有素材能提供哪种真实证据？
3. 哪条机制能把标题、封面、正文、评论与对应指标接成一条链？

检索键固定为：

```text
traffic_stage × primary_job × carrier/task_fit × available_real_materials
```

只允许放宽 `category`；不得为了“有结果”而换掉 `primary_job`、真实素材、授权、事实或合规约束。没有匹配机制时，只能交付 `candidate_only / needs_review` 和补证计划，不能称“可直接发布”“已验证流量”或“爆款公式”。

机器选择不是“按分数取前 5”。每次必须组成且只组成三槽：

```text
1 content
+ 1 carrier_router | truth_gate
+ 1 feedback | measurement | governance
```

19 张卡都有 `mechanism_kind / required_material_codes / forbidden_material_codes / requires / conflicts_with / activation / priority`。选择器先校验载体真实性条件和素材，再在每槽各取一张；素材不齐返回 `needs_materials`，某槽没有任务适配卡才返回 `needs_research`。因此 TM14—TM19 这类反馈、测量与治理卡不会因为“所有 job 都兼容”而挤满结果。

```bash
python scripts/select_traffic_mechanisms.py \
  --stage save_share \
  --job decision_support \
  --carrier comparison_warning \
  --materials comparison_candidates,uniform_protocol,proof_inventory,first_party_metrics,comment_semantics \
  --json
```

`materials` 是已存在且已核验的生产输入，不是希望模型生成的东西。传入 `fabricated_evidence`、`paid_organic_mixed` 等禁用码会阻断对应卡；未知码会直接报错，防止拼写错误静默放行。

## 2. 证据与指标分层：先知道每种材料能证明什么

JSON 的 `evidence_layers` 与每个 `source_ref.evidence_layer` 使用同一组 ID；来源可核验度与主张强度分开判断。官方主体的历史课程仍可能只是 `B_contextual`，因为它只能证明“平台当时这样教”，不是当前规则。

| `evidence_layer` | 可以支持 | 不能支持 |
| --- | --- | --- |
| `first_party_course_instruction` | 指定课程、日期和对象内的生产教学与任务区分 | 当前规则、自然算法权重、固定生命周期 |
| `production_artifact` | 对应记录范围内岗位、Brief、供给、劳动、治理或测量流程的存在 | 流程本身带来流量；岗位能力等于具体执行 |
| `academic_study` | 论文样本、方法和任务内的观察与审计变量 | 跨时期/平台/类目的普遍因果 |
| `brand_or_paid_case` | 项目披露范围内的素材、筛选、投放和复盘对象 | 把混合结果归因给一个标题/封面；付费胜出外推自然 |
| `operator_course_experience` | 查询方向、字段设计和单账号待测假设 | 平台事实、稳定效果、万能句式 |
| `live_public_proxy` / `live_public_proxy_audit` | 补采对象、`task_fit`、`series_constant`、反例和匹配对照 | 曝光、CTR、推荐权重、自然流量因果或机制晋级 |
| `mixed_secondary_evidence` | 定位原子来源和形成研究问题 | 绕过原始来源核验或升级证据等级 |

所以：

- “小红书官方有这个能力”不等于“小红书自然流会奖励它”。
- “品牌案例增长”不等于“某个内容动作造成增长”。
- “公开赞藏评高”不等于“曝光高”，也不等于“精准流量”。
- “样本里反复出现”只够提出假设；它必须带失败条件和反例。

每张卡的指标固定拆成三本账，不能再用一个 `primary` 字段混写：

| 账本 | 只记录什么 | 缺失如何处理 |
| --- | --- | --- |
| `traffic_exposure` | 本账号后台在明确 `surface/window` 下提供的 `impressions/reach` | 写 `not_observed`，不从公开赞藏评、搜索位置或第三方估算反推 |
| `job_outcome` | 发布前声明的一个 `primary_job` 事件与分母，如有效打开、读完、目标语义问题解决或主页延续 | 不用曝光代替，也不在看到结果后换口径 |
| `diagnostics` | 首屏退出、未兑现、注意力泄漏、来源未知、版本/权利缺口等故障定位 | 只解释哪一环可能坏了，不单独写成流量结果 |

## 3. 先选环节，再选机制

| traffic stage | 用户此刻的判断 | 优先机制 |
| --- | --- | --- |
| `feed_stop` | 这与我有关吗，值得点开吗 | TM01 承诺—证据距离；TM03 用户语言×场景；TM04 变化合同；TM18 入口路由；TM19 话题机会闸门 |
| `read_through` | 它会不会兑现，是否可信 | TM02 钩子兑现；TM09 真实使用证据；TM11 共同阻力到实际回报；TM12 问题教育 |
| `save_share` | 以后能不能拿来用、帮我做选择 | TM05 真实工作物；TM06 比较轴；TM07 搜索意图簇 |
| `comment_cocreation` | 我能补充什么、有什么条件或反例 | TM13 评论语义回声；TM14 有效爆款门 |
| `profile_follow` | 这个账号以后是否持续有用/有趣 | TM10 角色/系列承诺与新鲜负载；TM17 血缘与一致性 |

TM01 是入口内容机制；TM02 是全文兑现真实性门，不能被选择器当成第二条主内容。TM08 路由载体，TM18 路由推荐发现/搜索承接，TM19 在写作前判断话题是否值得进入候选池。TM15 是实验/复盘循环，TM16 是 AI 下游质量门；这些路由、测量与治理卡都不应被包装成单篇钩子。

## 4. 载体按任务选，不按“爆款审美”选

| primary job | 更常见的合适载体 | 必须提供的证据 | 常见错配 |
| --- | --- | --- | --- |
| `search_answer` | 截图标注、步骤清单、真实工作物、路线/答案地图 | 当前版本、路径、来源、边界 | 只做漂亮海报，不给动作或答案 |
| `decision_support` | 比较板、产品/方案合照、协议表 | 候选范围、统一比较轴、取舍 | 红黑榜有了，但标准随结论变 |
| `trust_build` | 真实过程、使用日记、变化序列 | 使用关系、过程、限制、失败点 | 故意低清/杂乱冒充真实 |
| `relationship_build` | 生活档案、系列代理视角、真实故事 | 时间/事件、选择与后果、真实性标签 | 用营销号总结代替事件 |
| `explain/authority_statement` | 文档、白底编辑、图解、真实界面 | 范围、来源、当前性、答案地图 | 深蓝、大字、机构感冒充权威 |
| `conversion` | 场景证据、选购标准、真实体验 | 产品事实、价格/区域/库存、适合与不适合、披露 | 先用无关剧情抢互动，再硬插产品 |

视频也是一等载体，而不是图文的附注：

- `process_video`：需要可授权真实过程或变化序列；演示/补拍必须标明。
- `screen_recording`：需要当前界面、版本/日期与隐私遮罩。
- `talking_head_or_field_video`：需要真人/场地授权和口播主张依据；资历与经历可追溯。
- `chat_dramatization`：只能二选一——授权的真实聊天原始记录，或在画面/正文显著披露的虚构演绎；“像聊天记录”不能伪装成真人投稿。

“实拍、真人、PPT、便签、手写、大字、网格、低饱和”都不是默认加分或默认禁用。判断标准只有一个：它是否完成当前任务，并且证据真实、可用、可授权。

## 5. 19 张机制卡

### TM01 承诺—证据距离

**公式**：唯一承诺 → 首屏第一份真实证据或答案地图。它只负责入口，减少用户从“看懂题目”到“确认这篇有料”所需的认知跳步。

- 标题子配方 OX06：先写无修辞事实标题并核对对象、场景、数字、期限和结果；再且只再选一种压缩——信息差、具体痛点、有边界情绪或直接答案。数字/符号只在降低歧义时使用。
- 封面：能给证据就给证据；需要延迟揭晓时，先给对象范围、结果差或答案地图。
- 正文：首段/第一页开始交付，后页持续关闭同一承诺。
- 评论：记录未兑现、条件和反例，不靠争议评论修补错误钩子。
- 看数：一方 `impressions/reach → CTR/open → 首屏停留/读完`；公开评论只能看是否复述核心矛盾或指出标题党。

证据：当前规则边界只有在回查同作用域官方原页时可按 A 处理；2024 商家课是 `B_contextual`，品牌项目是 C，站内样本只是 `public_proxy_association`。这些材料支持生产审查，**不支持“证据更近就一定高流量”**。大字、箭头和数字只能帮助定位，不能替代证据。

### TM02 钩子兑现契约

**公式**：把标题/封面的每个承诺映射到正文或逐页的明确兑现位置：`promise → proof → payoff → boundary`。它负责全文兑现，不是第二个标题/封面钩子。

- 标题/封面回答同一人群、场景和任务。
- 正文逐页新增信息；只重复封面的页面合并或删除。
- 结尾给适用边界，不用与前文无关的产品突袭。
- 评论中的“没讲答案、求具体步骤、标题党”编码为未兑现点。

这不是固定“痛点—方法—总结—互动”四段式，也不规定页数。它要求的是承诺闭合，不是统一版型。

### TM03 用户语言 × 场景 × 单一任务

**公式**：产品/专业事实 → 某类人在具体时刻能看见的麻烦或期待 → 一篇只解决一个任务。

输入优先级：当前搜索词与联想 → 真实评论/客服 → 访谈/试用 → 场景观察 → 产品事实与限制。没有这些输入时，不凭模型常识编造“用户原话”。

- 标题先写用户会识别的任务，不照搬品牌大词。
- 封面先让用户认出“这是我的处境”，不要塞全卖点。
- 正文走 `处境 → 判断标准 → 事实如何作用 → 使用/执行 → 适合与不适合`。
- 评论观察用户是否自然复述目标问题，并把新表达回写词库。

反面：把“痛点”写得更惨、强造昵称、用夸张形容词替代产品事实，都不属于这条机制。

### TM04 变化合同

**公式**：真实 before + 可感知 outcome + 有意义的约束/代价 + 过程证据。

- 首屏的 before 要让原问题可见，并给出结果幅度、真实约束或翻页任务。
- 正文按 `起点 → 决策依据 → 步骤 → 结果 → 成本/维护/失败 → 边界` 关闭承诺。
- 评论回答预算、材料、尺寸、时间和失败条件。

租房样本中，高低公开代理都用了 before，所以 before 只是 `task_fit`。箭头、低成本和“左滑”也不是流量按钮。没有真实过程、健康功效证据或可复制边界时，不使用这条机制。

### TM05 真实工作物首屏

**公式**：对高意图任务，先展示用户未来会提交、操作、核对或带到现场的真实对象，再教怎么用。

适合：法律/财务清单、申请材料、软件设置、路线、维修、表单、设备操作等。硬门：版本、日期、辖区、来源、隐私和授权。

- 标题限定具体情境和版本。
- 封面让真实表、单、页面、截图或工具成为主体。
- 正文按“一步一定位”讲路径、动作和结果，并给错误示例。
- 出错时公开更正，不把关键答案藏进私信。

站内高低样本都出现真实文件，因此“打印纸、荧光笔、红框”都不是流量公式。工作物只说明载体与任务相容。

### TM06 比较轴与统一协议

**公式**：相同条件 + 多候选 + 明确选择轴 + 一致测试协议 + 分人群取舍。

- 先披露本人条件、候选范围、测试条件和商业关系。
- 封面让候选和比较轴可见；红黑、打分和网格只是编码方式。
- 每项至少给证据、优势、限制和适合谁；不制造唯一冠军。
- 评论征集不同条件、反例和下一批候选。

没有统一条件、标准随结论变化、只夸不贬、假专业身份或未披露赞助时，机制失效。数字标题必须与实际候选数一致，但数字本身不是钩子常数。

### TM07 搜索意图簇与答案地图

**公式**：品牌/产品词承接已有认知，品类/属性词承接比较，场景/人群词承接问题；一篇只回答一个主查询，系列完成意图路径。

- 每次运行都重新采集搜索联想、结果构成、评论问题和时间/入口。
- 标题自然包含主查询；“怎么选”就给标准，“怎么做”就给步骤。
- 封面预览答案形态：路线、比较、步骤、结果或对象范围。
- 首段直接回答，再展开依据、限制和相关问题。

搜索位置只代表当前入口、时间和账号环境。关键词堆叠、旧报告热词、固定密度和“标题正文图片话题都埋词”不属于本机制。

### TM08 载体按任务与证据选择

**公式**：`primary_job × available evidence × relationship goal → carrier`。

简表：

- 清单：给标准和取舍；
- 改造：给起点、过程、结果和代价；
- 教程：给路径、动作与坑；
- 比较：给协议、候选与分支结论；
- 关系故事：给事件、选择、后果和时间证据；
- 正式解释：给范围证明、答案地图和来源。

每页只做场景、证据、步骤、比较、转折、结论或边界中的一件事。图片与 caption 分工，不逐字复述。所有内容都做成 PPT、手写便签或随手拍，都是错配。

### TM09 真实使用证据 × 有边界的不完美

**公式**：真实使用关系 + 可见过程/消耗/状态 + 具体优点 + 具体限制。

- 标题用亲历任务或有边界判断，不伪装路人自购。
- 封面可以手持、拆开、过程、环境痕迹或使用后状态，但不强制低清和凌乱。
- 正文回答为什么需要、怎么用、哪里好、哪里麻烦、谁别用。
- 商业关系正常披露；二次投放另有授权。

原生感不是粗糙度。故意加噪点、杂物、错别字、低饱和或“素人滤镜”都无法替代真实关系和具体取舍。

### TM10 角色/系列承诺 × 稳定签名 × 新鲜负载

**公式**：可信角色或稳定代理视角 + 明确系列承诺 + 可识别签名 + 每篇新事实/新情节/新工具。身份张力只在真实且能帮助理解时可选，不是系列成立的必填钩子。

- 角色负责解释权，系列承诺说明读者下次还能得到什么；不能冒充资历。
- 标题先兑现本篇新负载；只有真实反常问题能帮助理解时才用张力，不能制造恐慌。
- 封面保留少量识别常量，但本篇独特矛盾或证据必须更突出。
- 正文至少增加一项新阶段、事实、工具或选择。
- 评论允许读者在同一语义场补真实经验和反例。

同账号同模板公开表现仍可能差很多，说明固定大字、代理词、配色和长文是 `series_constant`，不是流量按钮。不要复制别人的身体部位、拟人代理、口头禅或封面。虚构代理必须显著披露；无新负载时不应只靠系列签名发一篇。

### TM11 共同阻力 → 可行动的更好生活

**公式**：真实共同阻力 + 可见转折/选择 + 目标生活画面 + 可执行方法或工具。

- 标题写具体阶段和阻力，不先写产品或空泛价值观。
- 封面用场景、人物/物件或冲突证据建立代入。
- 正文走 `阻力 → 失败尝试 → 选择/转折 → 结果与代价 → 可复用方法`。
- 情绪入口和实用出口必须来自同一场景。

没有真实/授权经历时，降级为经验清单或显著虚构，不伪造投稿、聊天或第一人称。高意图搜索任务可能更需要工作物或清单，不必强行故事化。

### TM12 陌生类目先问题教育

**公式**：用户已感知的问题 → 旧办法为何不够 → 新机制与选择标准 → 产品/品牌作为一个选项。

- 标题写问题或类目任务，不把陌生品牌当第一钩子。
- 封面给问题场景、结果差或选择地图。
- 正文分开问题教育、机制事实、判断标准、产品实例和不适用条件。
- 评论用于核对用户是否理解机制，不制造焦虑推动成交。

它不是“在广告前加一段科普”。如果用户已经在搜明确购买词，就应直接做选择/比较任务，而不是重复教育。

### TM13 评论语义回声与真问题闭环

**公式**：正文提出可回答的真实问题 → 评论复述/补充同一语义 → 公开回应与更正 → 下一篇选题。

- 发布前定义希望出现的 1—2 类真实问题。
- 首屏吸引的语义与目标评论应一致，避免颜值/争议截流。
- 回答疑问、补条件、承认不确定、征集反例。
- 把高频真实问题改写成下一篇，不伪造“网友都在问”。

主指标是目标语义占比、追问深度、问题解决/更正和主页延续，不是评论总量。评论区不是分发机关；不做马甲号、预埋评论、固定条数、置顶争议、私信暗号或竞品评论区截流。

### TM14 有效爆款门与注意力泄漏

**公式**：表面表现通过 + 目标语义通过 + primary_job 对应的后续行为通过，才允许进入复制或放大池。

逐项检查：

1. 表现是否相对账号内同任务基线异常，而非跨账号绝对高；
2. 用户讨论的是目标问题/产品，还是外貌、剧情、抽奖、争议或其他产品；
3. 搜索、主页、商品或业务动作是否与主任务一致；
4. 自然、活动、平台邀约和付费是否分账。

高赞但评论都聊颜值/剧情，就是注意力泄漏。相关评论也不等于成交。统一“千赞爆文”、互动率门槛和“爆了就复制”都被拒绝。

### TM15 候选小测 → 阶段诊断 → 单变量资产循环

**公式**：同一任务下做可比较候选，按当前漏斗瓶颈只改一个声明变量，记录 `win/loss/inconclusive`、反例和适用边界。

- 候选必须是不同注意力路径，不是只换颜色。
- 测标题时保持任务/答案稳定；测封面时保持正文核心交付稳定。
- 停留差看选题/标题/封面，读完差看证据和节奏，收藏差看复用价值，关注差看系列/主页，商业动作差看人群×场景×需求与承接。
- 保存失败和淘汰原因，不只收藏成功链接。

自然发布只能叫准实验，两篇帖子不是随机 A/B。付费赛马必须先锁目标、事件、窗口、去重、退款、基线、固定变量、版本变量和停止条件。没有固定的“每个选题做 3 个封面”数量规则。

### TM16 AI 初稿下游修复

**公式**：AI 草稿 → 事实/时效 → 来源 → 真人确认 → 用户语言 → 叙事因果 → 合规 → 镜头可执行性。

- 标题核对数字、时间、来源和趋势；删除模型补出的经历与伪引语。
- 封面确认人物/场景与正文同源；AI 图不充当体验或效果证据。
- 正文标出 `source_fact / human_experience / brand_requirement / model_suggestion`，分别处理。
- 视频让真实创作者试读/试演；文字顺不等于镜头能完成。
- 保存 AI 草稿、人工修改和最终版差异。

访谈研究支持“AI 后仍有大量下游劳动”，不支持“人工改写会提升流量”。加 emoji、口语词、错别字、手写体或同义词替换，不等于去 AI。

### TM17 证据—供给—版本血缘闸门

**公式**：每条结论和数字 = 来源 + 供给类型 + 素材/发布版本 + 自然/付费 + 采集时间 + 观察窗 + 指标口径。

最低记录：

```text
primary_job / evidence_assets / rights / owner / acceptance
sample_id / author_id / query / surface / captured_at
supply_source = spontaneous | brand_campaign | platform_program | paid_distribution | unknown
capture_id → rendition_id → publish_id
metric_definition / data_source / refreshed_at / observation_window / attribution / paid_flag
```

- 活动、平台邀约和商业投放都可能制造内容密度；100 篇同 brief 不能当 100 个独立需求。
- 替换标题、封面、正文或标签要生成版本事件，不覆盖旧值。
- 后台、竞品公开代理和第三方工具分表；超出工具追踪窗口标 `not_observed`，不能填 0。
- 官方页面只证明其明确作用域，不自动成为“官方流量机制”。

这条机制不提升流量；它是任何流量结论能够被相信、复算和降级的前提。

### TM18 获客入口路由：推荐发现 × 搜索承接

**公式**：`当前入口证据 × 用户当下状态 × primary_job → recommendation_discovery | search_resolution | dual_entry`。

- 推荐发现：先让未主动搜索的人识别身份、场景、变化或证据缺口，再兑现一个承诺；不能用无关争议抢停留。
- 搜索承接：使用自然的实体—问题—场景—答案表达，标题与封面预览对象范围、答案形态或操作路径；不能堆同义词。
- 双入口：只在同一正文能完整服务两种任务时共用；否则保留同一事实底稿，分别重写标题、封面和首段。
- 评论：将真实问题标成 `discovery_question / active_search_query / condition / counterexample`，回写下一轮入口观察。

硬输入是 `surface_entry_observation + proof_inventory`。前者必须带 `captured_at / surface / query_or_entry / account_environment / paid_flag`；没有后台来源就写 `unknown`，不能由搜索位置或公开互动补出。主来源 OX04 是 **2024 年商家公开课**：它支持当时的任务区分，不支持 2026 当前权重、推荐/搜索比例或固定寿命。自然与付费必须分账。

### TM19 话题机会闸门：需求 × 供给 × 时效 × 证据

**公式**：`独立需求信号 × 可审计供给缺口 × 当前时效 × 可交付真实证据 → topic_candidate`。这不是平台打分公式，而是开写前的证据门。

1. **需求**：跨作者关系、入口或时间重复出现的具体问题；按语义去重，不把转载和同一 Brief 重复计数。
2. **供给**：逐样本标 `spontaneous / brand_campaign / platform_program / paid_distribution / unknown`；同模板与活动标识是供给线索，不是审美偏好证明。
3. **时效**：记录 `last_verified_at`、事实变化和失效条件；没有固定天数，“新发”也不等于信息新鲜。
4. **证据**：本账号有什么真实、可授权的事实、经验、过程、工作物或比较能补上答案缺口。

标题取一个独立用户任务，不复制活动口号；封面让本账号独有证据成为主体；正文按 `需求场景 → 现有答案缺口 → 本次证据 → 可执行答案 → 边界` 交付；评论只把新问题记为 `demand_candidate`，不立即宣称“大家都在问”。任一项 `unknown` 就返回补采或降级。HP04 证明供给来源值得审计，**不证明四因子会提升自然流量**。

## 6. 推荐组合，不是公式堆叠

每篇固定选择三槽，不按建议数量无限叠加：

```text
1 content
+ 1 carrier_router / truth_gate
+ 1 feedback / measurement / governance
```

例如：

| 任务 | 内容机制 | 载体/真实性 | 复盘/治理 |
| --- | --- | --- | --- |
| 软件设置教程 | TM05 工作物首屏 | TM08 录屏/截图载体路由 | TM17 版本/来源 |
| 多产品测评 | TM06 比较轴 | TM09 真实使用证据 | TM14 有效爆款门 |
| 租房改造 | TM04 变化合同 | TM08 真实照片序列 | TM15 单变量循环 |
| 个人成长系列 | TM10 角色/系列承诺 | TM09 真实经历证据 | TM13 评论闭环 |
| 新功能/新品类 | TM12 问题教育 | TM02 全文兑现门 | TM17 来源/版本 |
| AI 辅助成稿 | 由主任务决定 | TM16 下游修复 | TM17 来源/版本 |
| 推荐发现后由搜索承接 | 由具体任务决定 | TM18 入口路由 | TM17 入口/版本分账 |
| 热点、趋势或竞品爆文选题 | 由具体任务决定 | TM08 载体待定/按证据选择 | TM19 话题机会闸门 |

不要一次同时上“身份冲突、反常识、数字、焦虑、前后对比、悬念、争议”七个钩子。钩子越多，承诺越难兑现，注意力越容易偏离。

## 7. 发布前最小检查

1. 本篇唯一 `primary_job` 是什么？
2. 标题、封面和第一屏是否在回答同一任务？
3. 第一份真实证据在哪里，是否授权、当前、准确？
4. 载体是否真的适合这个任务，还是只因为参考帖高赞？
5. 正文是否给过程、标准、取舍或边界，而非只给形容词？
6. 希望出现什么真实评论；无关注意力怎样标记？
7. 主指标是什么；公开代理与一方数据是否分开？
8. 自然、平台活动、品牌合作和付费是否分账？
9. 这一版只改变什么；失败后只改哪一环？
10. AI 参与时，事实、来源、真人经验和镜头是否完成下游修复？

任何关键项答不出来，就先补素材、研究或事实，不用生成更多“像小红书”的装饰。

## 8. Source ref 命名空间

JSON 中的 `source_refs` 使用命名空间而不是裸 `S01/P01`，避免不同研究文件编号冲突。每个 ref 都带 `evidence_layer / scope / limitation / role / ref_type / locator`；`OX03/OX07`、`S2/S4` 这类含糊复合写法禁止进入机器资产。

`ref_type` 只允许四种：原子记录 `atomic_record`、标题节 `section_anchor`、节内编号项 `section_item`、明确的聚合结论 `composite_summary`。后三种必须写到具体文件与 `#anchor`；聚合结论只能用来找回组成来源，不能升级证据等级。

| --- | --- |

## 9. 当前限制

- 这批资料没有提供一个可证明、跨类目、稳定的“爆款公式”。
- 站内实时样本仍以公开互动代理为主，账号体量、帖子年龄、投放、外部分发和完整页面经常无法控制。
- 付费/品牌案例更适合还原生产链路，不适合解释自然算法。
- 当前 19 张卡没有任何 `first_party_traffic_validated` 机制；首次使用必须结合具体账号、类目、载体、入口和一方数据继续校准。
- 机制资产允许迁移，原句、图片、人物、独特构图、品牌资产和作者代理 IP 不允许复制。
