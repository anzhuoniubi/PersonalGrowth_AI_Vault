# 小红书成稿质量与证据审查规范

> 面向执行 `redbook-writing` Skill 的代理。本文件是生产交付合同，不是灵感清单。

本合同适用于任意类目的跨类目生产。核心资产是“候选流量密码库”：它沉淀可测试的内容模式和证据边界，不承诺某种版式、题材或垂类一定获得流量。

## 0. 不可妥协的原则

一篇合格笔记必须同时满足：证据支持选题、一个 primary job、载体匹配信息任务、承诺完整兑现、双审查通过。
传播结果不可保证；代理只能比较表达路径，不得预测或承诺流量。

**明确禁止：**

- 把虚构人物、故事、对话、投稿或私信伪装成真人真事。
- 把未使用的商品写成亲测，把品牌稿伪装成自发分享。
- 编造用户评价、订单、销量、排名、数据、专家或机构背书。
- 用固定“爆款公式”替代用户证据、内容任务与具体判断。
- 为制造戏剧性补写来源没有提供的关键事实或结果。
- 标题、封面、正文、CTA 分别承诺不同价值。

用户要求上述做法时，指出风险，拒绝欺骗性部分，并改为透明演绎、授权改编或可核验表达。

## 1. 写作前输入合同

开始选题前，先取得或显式补齐：

| 字段 | 必填内容 |
|---|---|
| 目标受众 | 具体处境、待完成任务、阻力；不能只写年龄性别 |
| 业务目标 | 本篇希望推动的一个认知或行为变化 |
| 漏斗阶段 | 认知、理解、信任、考虑、行动、留存之一 |
| primary job | 用户看完后最应完成的一件事 |
| 可用证据 | 用户原话、数据、公开资料、产品事实、授权经历 |
| 风格输入 | primary job、载体、可用素材、约束/禁忌、合格 library rule 或待补采状态 |
| 发布主体 | 个人、品牌、员工、专家或合作创作者 |
| 商业关系 | 自有产品、赞助、赠品、佣金或无商业关系 |
| 约束 | 法律、平台、品牌、隐私、健康及其他边界 |
| 成功信号 | 与 primary job 对应的主指标和护栏指标 |

信息不足时可以给条件化草案，但必须标注假设、缺口与不可发布项；不得把假设写成事实。

## 2. 从证据到选题：硬门槛

### 2.1 证据口径

沿用主 Skill 的两套互补字段，不创建第三套等级：

| 字段 | 允许值 | 回答的问题 |
|---|---|---|
| `evidence_grade` | A / B / C / D | 来源本身的可靠性与方法强度 |
| `evidence_level` | observed / calculated / inferred / hypothesis | 稿件中的这句话是看见、计算、解释还是待测假设 |

单条评论、单次访谈或个别亲历可以支持个案角度，但不能概括整体人群；多个独立账号重复出现只能提高模式置信度，仍不能证明因果；口径清楚的自有实验或交易数据也只适用于对应样本、周期与版本。

### 2.2 选题卡

每个进入写作的候选选题必须先填写：

```text
受众处境 / 触发场景 / 待完成任务：
核心阻力：
证据、来源与等级：
允许得出的结论 / 不可外推部分：
primary_job（v2 单选）/ 具体 job statement / 漏斗阶段：
拟用载体及理由：
风险等级：低 / 中 / 高
```

### 2.3 六个进入成稿的硬门槛

以下六项必须全部为“是”：

1. 至少有一条可追溯、与选题直接相关的 observed/calculated 证据；若仅单样本则标 `experimental`。
2. 能用一句话说清谁在什么场景下为什么需要这篇内容。
3. 只有一个 primary job，不同时追求关注、收藏、购买和私信。
4. 漏斗阶段明确，并能解释为何不是相邻阶段。
5. 正文有足够内容兑现标题，不只有情绪钩子。
6. 不靠编造身份、经历、测评、效果、数据或社会证明也能成立。

任一项为“否”：**STOP。** 退回研究、补证据或重构选题，不写成“可发布成稿”。

`experimental` 不是“可以无证据自由创作”。它仍须绑定至少一条可追溯的需求、评论、访谈或内容样本，并明确只是待测角度。零样本时只能输出研究问题和查询候选，不能输出选题、标题、封面或成稿。

### 2.4 风格证据的 ready 门与候选生产

写稿前按 `category × carrier × primary_job × materials × constraints × contraindications` 查询已发布、未过期的 library rule。若没有合格 rule，只有完成真实站内 qualified-cell 研究后才可检查 starter；当前门禁若为 `qualified_cells=0`，禁止创建、使用或宣称 `starter_applied`。

Library 和 starter 都不合格时设置 `style_binding_status=needs_style_research`，**停止 ready 生产**，但不要把“证据诚实”变成“什么也做不了”。只有同时满足下面四项，才允许继续做 `candidate_only / needs_review`：

`candidate_only` 不是自由创作通道：三槽 task-fit 机制、独立反例/anti-pattern 和真实素材凭据缺一不可。

1. 运行机制选择器并绑定恰好 3 条与当前 `traffic_stage × primary_job × carrier × available materials` 相符的机制：1 条内容、1 条载体/真实性、1 条复盘/治理；
2. 同时绑定至少 1 条反例或 anti-pattern，说明哪些表面元素不能照搬；
3. 事实、授权、真实性与商业门已满足；缺少真实证明素材时，改选不冒充证据的载体；
4. 输出显著标注为探索候选，并附 exact 风格补采 query、缺口和最小验证计划。

候选路径可以交付 2–3 个标题假设、完整候选正文/逐页结构和结构性视觉 brief；存在两个各自有依据、注意力路径真正不同的合格方向且素材权利允许时，可交付两个粗原型。只有一个合格方向时最多交付一个明确标注的 `prototype_only`，不得声称完成比较或扩成 final；没有合格方向才标 `prototype_gap / brief_only`。不能为满足数量临时发明第二种审美。候选只能标 `needs_review`，视觉只能到 `prototype_only`；不得称 ready、可直接发布、已验证流量或爆款公式。若连 task-fit 机制、反例或真实素材条件也不足，才完全停止标题/正文/图像，只交付研究缺口。

这条候选路径不是“先生成再补证据”。在 draft 的 `## 流量机制绑定` 使用机器合同：`contract_status / primary_mechanism_id / mechanism_ids / counterexample_ids / material_codes / material_evidence_map / mechanism_application_map / research_gap`。每个 required material 必须指向本次运行里真实存在的 Source / Claim / Post / Authorization / material ID；逐机制写清标题、封面、正文、评论动作、job metric、从原卡原样选择的失效条件和有理由的偏离。假 ID、同一 ID 兼任支持与反例、job/carrier/stage 错配或三槽不齐都会阻断。无引用的花字、配色、版式和口头禅一律删除。

### 2.5 直接淘汰

- 唯一依据是“最近大家都在做”或“这个形式容易爆”。
- 需要冒充消费者、专家、路人、真实投稿者才能成立。
- 结论依赖无法证明的功效、收入、成功率、排名或绝对比较。
- 需要未经授权披露聊天、头像、位置、病史、关系或订单信息。
- 主题不适合未成年人，却用未成年人作受众或素材。
- 内容价值只能靠“扣 1、私信暗号、求链接”维持。

## 3. Primary job 与漏斗

### 3.1 单一 primary job

V2 `primary_job` 只允许：`feed_stop | search_answer | explain | trust_build | decision_support | relationship_build | conversion | authority_statement`。另写一句具体 job statement，避免只填枚举而没有用户任务。旧值 `recommendation_reach | search_capture | relationship_building | commercial_conversion` 只属于 legacy v1 run；v2 不静默映射旧值，也不靠自由文本创造近义 code。

| v2 code | 单篇首要任务 |
|---|---|
| `feed_stop` | 让目标用户在信息流识别“这与我有关” |
| `search_answer` | 对明确搜索问题先给可定位答案 |
| `explain` | 解释原因、差异或误区 |
| `trust_build` | 展示证据、过程与边界 |
| `decision_support` | 提供比较标准和取舍 |
| `relationship_build` | 建立持续栏目关系与身份共鸣 |
| `conversion` | 在资格与披露通过后降低一个行动阻力 |
| `authority_statement` | 发布正式、可核验的机构或品牌说明 |

### 3.2 跨类目 traffic funnel

`primary_job` 回答“这篇内容首先帮用户完成什么”，`traffic_funnel_stage` 回答“当前要诊断漏斗哪一段”，两者不得混成一个字段。默认 funnel 枚举：

```text
feed_stop → read_through → save_share → comment_cocreation → profile_follow
```

- `feed_stop`：目标用户是否在信息流识别并停留；一方 traffic verdict 仍只用 impressions/reach。
- `read_through`：打开后是否继续阅读/观看；完读、停留仅作 attention diagnostic。
- `save_share`：内容是否值得回看或转给具体对象。
- `comment_cocreation`：评论是否补充问题、反例、经验或下一选题，而非只凑互动。
- `profile_follow`：用户是否进入主页并建立后续栏目关系；记录主页访问与关注率护栏。

每稿只选一个当前 funnel stage 和一个 primary job；后段指标不能倒推前段曝光，也不能用公开互动替代一方流量。

指标必须分两条：

- `traffic_primary_metric` 只回答“获得了多少实际 exposure”：一方 impressions，平台只给 reach 时固定用 reach；没有就 unavailable。
- `job_primary_metric` 回答“打开以后是否完成这篇的主要任务”：例如 search answer 的答案完成/保存、decision support 的比较完成、relationship build 的主页访问/关注。必须写事件定义、分母、数据范围和护栏。

公开竞品没有 exposure，也通常没有 job 分母；最多记录 `comment_semantic_proxy`，verdict 固定 `not_applicable`。不得用收藏、完读、CTR、关注或成交顶替 exposure，也不得用曝光上升证明任务已完成。

使用句式：

> 让【具体受众】在【场景】完成【一个可观察的认知或行为变化】。

合格：“让第一次买防晒衣的通勤用户，用三个参数排除不合适款式。”
不合格：“提升声量、促进互动、教育用户并带动销量。”
后者是多个业务愿望，不是单篇内容任务。

### 3.3 漏斗匹配

| 阶段 | 用户问题 | 常见 primary job | 常见主指标 |
|---|---|---|---|
| 认知 | 这和我有关吗 | 命名具体场景与问题 | 目标人群有效观看 |
| 理解 | 为什么会这样 | 解释机制、差异、误区 | 完读、有效收藏 |
| 信任 | 你凭什么这样说 | 展示证据、过程、边界 | 资料访问、质询质量 |
| 考虑 | 哪个方案适合我 | 给选择框架与取舍 | 对比行为、真实意向 |
| 行动 | 下一步怎么做 | 降低执行阻力 | 合规点击、咨询、转化 |
| 留存 | 如何持续获得价值 | 教复用、维护或进阶 | 复访、复购、长期反馈 |

认知内容不得突然使用行动阶段的强销售 CTA；考虑内容不得用泛情绪故事替代参数与取舍。
收藏、评论、关注可为次要收益，但不能争夺正文结构；删掉 CTA 后内容仍须独立有用。

## 4. 载体按信息任务选择

先判断用户需要怎样理解，再选载体：

| 信息任务 | 首选载体 | 选择理由 | 停止信号 |
|---|---|---|---|
| 还原选择或关系变化 | 故事 | 人物、因果、选择承载变化 | 无真实事件，只能编细节 |
| 展示措辞、分歧与边界 | 聊天记录/对话演绎 | 逐轮观察语言影响 | 核心其实是知识清单 |
| 比较步骤、参数、误区 | 轮播 | 分层、对照、便于回看 | 每页只是同义改写 |
| 传递单一结论或提醒 | 单图/短文 | 认知负担最低 | 需要解释机制或证据 |
| 展示动作、质地、空间、过程 | 视频 | 动作与时间本身是证据 | 画面只是朗读字幕 |
| 提供复用工具 | 清单/模板轮播 | 便于保存、执行、复查 | 工具没有适用边界 |

写明：“选择【载体】，因为用户需要【信息任务】，不是因为它更容易爆。”
两种载体都可行时，选择信息损失更小、制作成本更可控者。

## 5. 故事、聊天记录、轮播的专门门槛

### 5.1 故事门槛

故事必须有主角、处境、目标、阻力、选择、结果，并满足：

- 至少一个由选择造成的变化，不用巧合代替因果。
- 区分已知事实、当事人感受与作者解释。
- 结尾直接服务 primary job，不强行升华。
- 来源、真实性标签、关键事实均已确认。
- 不省略会改变读者判断的重要背景。

新增来源未提供的关键情节、改变因果的时间线、把个案写成普遍规律，均为 FAIL。

### 5.2 聊天记录门槛

聊天只能属于三类，并在首屏或首段标注：

1. **本人真实记录：** 发布者拥有记录，相关方已授权。
2. **授权匿名改编：** 基于真实记录，已授权，改动不改变核心事实。
3. **情境演绎：** 人物与对话虚构，用于解释方法或典型场景。

同时满足：

- 每轮推进信息、关系或选择，删除填充轮次。
- 说话人可辨，不依赖真实头像、昵称或仿冒平台水印。
- 时间戳只在影响理解时保留，不用时间戳制造真实感。
- 省略内容不扭曲原意，关键上下文不藏在省略号后。
- 一屏一个主要转折，常见手机尺寸可读。
- 不浪漫化羞辱、胁迫、伤害或高风险行为。

禁止伪造“粉丝投稿、后台私信、闺蜜聊天、客户反馈”；禁止搬运截图后换头像、裁水印或改字。

### 5.3 轮播门槛

轮播页面依次承担：封面承诺、场景识别、原因/证据、步骤/对照、边界、自然下一步。
页数可增减，但每页只能有一个主要动作或观点，页标题可独立理解，顺序不可随意互换。

以下为 FAIL：

- 为凑页数把一句话拆成多页。
- 封面承诺清单，正文只有个人感受。
- 每页信息同权，重点无法识别。
- 末页突然出现与正文无关的商品或关注请求。
- 结论必须依赖小字免责声明才不误导。

## 6. 真实性标签与授权

### 6.1 唯一真实性词典

`truth_label` 使用机器值，正文首屏/首段再显示自然中文：

| truth_label | 建议展示词 | 使用条件 |
|---|---|---|
| `first_person_documented` | 本人亲历记录 | 发布主体确实亲历，关键事实可核验；可见标签必须说明“本人/我的亲历”，涉及他人材料还需公开或逐字权限 |
| `authorized_anonymized` | 经授权匿名整理 | 当事人同意发布与匿名处理，未改动核心事实 |
| `authorized_adaptation` | 基于真实经历改编 | 已授权并公开改编范围，不改变核心因果 |
| `composite_cases` | 多案例合成 | 来自多份合法材料，不对应单一真人，明确合成性质 |
| `fictional_scenario` | 情境演绎/明确虚构 | 人物与对话虚构，只用于解释方法或场景 |
| `factual_explainer` | 事实科普 | 以可追溯资料说明事实，不暗示真人故事 |

不得用“如有雷同纯属巧合”代替 `fictional_scenario`；不得以“保护隐私”为由隐瞒内容其实是虚构的。真实性标签必须是肯定式可见文字；HTML 注释、隐藏标签、图片 alt、链接 URL 与否定句都不算披露。

商业关系不属于真实性标签。另填 `commercial_relationship`：`none`、`owned_product`、`sponsored`、`gifted`、`affiliate`、`commissioned_creator` 或 `other_disclosed`。非 `none` 时必须把精确、肯定式 `disclosure_text` 写进实际 `## 成稿`，并让位置与 `disclosure_location` 一致；分别明确自有/自营、广告/品牌合作、获赠、佣金/返佣、受委托或其他利益关系，不能一边声明关系、一边写“不是广告/没有合作”。只填 frontmatter、合同块、HTML 注释或图片 alt 无效。真实性与商业披露两项可以同时成立，不能相互替代。

### 6.2 授权记录

真人经历、聊天、照片、声音、评价使用前，在 `authorization-log.csv` 建立 `AUTH-*` 记录；成稿 frontmatter 的 `authorization_ids` 必须引用它：

```text
authorization_id / subject_scope / material_type：
permission_scope（公开/改编/合成/逐字）与 commercial_use：
granted_at / expires_at / verified_at：
匿名与删改要求 / 撤回机制：
受控原件 locator / 核验人 / 最终衍生稿 SHA-256：
```

没有授权记录，默认没有授权；`authorized_anonymized`、`authorized_adaptation` 必须各有当前有效记录，`composite_cases` 至少两条且必须同时对应不同 subject、`material_id` 与 `material_sha256`，同一材料换 ID 或重复授权不计作多个案例；发布稿字节必须与授权记录中的 `authorized_output_sha256` 一致，修改后重新审查。范围不清即 PARTIAL，补齐前不得发布。
匿名化须覆盖姓名、头像、账号、位置、单位、时间、订单号、病史及可交叉识别细节。
涉及未成年人、健康、性、创伤、债务等高敏感信息，默认升级为高风险审查。

### 6.3 成人亲密内容的商业门禁

成人模块仅在内容命中相应敏感主题且同时触发 SKU、offer、商业关系、商业 CTA、付费分发或站外 destination 时启用；它不是 Skill 的默认类目或默认生产流程。

成人亲密关系、性健康或成人用品内容一旦同时出现商品、品牌合作、佣金、获赠、付费分发、站外去向或商业 CTA，必须命中 `adult_sensitive_commercial_gate`。在运行目录写入当前 `production-gate-receipts.jsonl`，逐稿绑定 exact SKU/offer、账号、surface、素材版本、授权与权利、商业关系披露、受众年龄边界、distribution/destination 和人工 reviewer。

Receipt 缺失、过期、字段 unknown、未成年人可达性未处理或 `gate_status != pass` 时，稿件必须 `blocked`，不得出现商品购买、私信索取、站外导流或广告投放动作。非商业科普仍需高敏感合规审查；商业门禁不替代真实性、授权、主张证明和平台规则核验。预注册首轮 12 帖实验不得夹带成人商品 CTA。

## 7. 标题、封面与承诺一致性

完整成稿提供 2–3 个标题/封面版本，用于比较表达路径，不称流量预测：

- A｜直接价值：明确用户得到什么。
- B｜场景张力：给具体矛盾，不隐去关键条件。
- C｜证据反差：突出可证明的差异、过程或结果。

每版都附：适合的受众状态、与 primary job 的关系、歧义或过度承诺风险。
禁止“预测流量最高、一定爆、算法最喜欢、点击率必然更高”。
若引用历史表现，必须限定样本、账号、周期、指标与不确定性。

先用 [cover-pattern-library.md](cover-pattern-library.md) 判断第一份证据由文字还是画面承担。`text-first job + visual_evidence_role=none/supporting` 时优先字卡；人物、场景、结果差、UI、文件、比较或质地是第一证据时优先相应证据型封面。不得为了统一主页视觉把真实证据藏到内页。

### 7.1 承诺一致性测试

```text
标题承诺：
封面承诺：
正文兑现位置：
证据支持：
CTA 下一步：
```

五项必须指向同一 primary job；任一承诺找不到正文兑现位置即 FAIL。
“实测”必须有真实测试过程；“避坑清单”必须有判断标准；时效承诺必须有时间和效果定义。
CTA 不得把标题承诺本应直接给出的内容扣留到私信，也不得把中性比较偷换成绝对推荐。

## 8. 完整成稿合同

用户要求“成稿、可发布、完整笔记”时，按顺序交付：

1. 写作判断：受众、primary job、漏斗、载体及选择理由。
2. 关键假设：仅列影响发布结论的假设与缺口。
3. 2–3 个标题/封面版本，各含选择理由和风险。
4. 最终推荐版本及理由，不称流量预测。
5. 可直接复制的完整正文，无占位符或“此处补案例”。
6. 故事、聊天或轮播按屏/页写完，并给必要视觉指令。
7. 与漏斗一致的 CTA、克制相关的话题词。
8. 实际展示的真实性标签与商业披露文案、位置。
9. 事实与功效证明账本。
10. compliance review 与 creative review 两份独立结论。
11. 自然准实验计划与生命周期动作。

只有内容完整、事实可追溯、标签已进入稿件、两项审查均 PASS，才能写“可直接发布”。
用户只要局部时可以缩短交付，但不能省略与该部分直接相关的真实性、证据和审查结论。

### 8.1 风格绑定与视觉生产闭环

风格不是形容词列表。只有 published binding 的 rule version、snapshot、association summary、`claim_kind` 和 `performance_evidence_scope` 完整闭合，才能把规则写入 brief。`public_proxy_association` 只表示公开互动对照下的候选关联，不是 traffic 验证；`series_constant/task_fit` 必须保持 `not_performance_evidence`。

视觉任务按以下合同执行：

1. 先从 `assets/cover-patterns-v1.json` 按 exact `primary_job × carrier × materials × visual_evidence_role` 选择封面模式。文字型任务且没有第一屏视觉证据时，字卡是条件默认；真实画面是证据时由证据型模式覆盖。记录 CP ID、拒绝模式和原因。
2. 再从 `assets/visual-direction-cards-v1.json` 按 exact `primary_job × carrier × required_materials × contraindications` 选择 1–2 张候选卡；记录 card ID、真实 prompt 变量、拒绝卡和原因。卡片固定为 `candidate_only/not_performance_evidence`，不构成 style rule 或 starter。
3. 无 exact published binding、但需要先看方向时，再从 `assets/aesthetic-exploration-prompts-v1.json` 按 exact `category_code × primary_job × carrier × direction_card_id` 选择最多 1 条 AP；同时核对素材数量、约束、权利、禁用项、source/prompt hash。只命中 analogue、research lead 或缺素材时停在 `brief_only`，不能跨格拼装。
4. 在 `visual-briefs.jsonl` 写不可变 brief revision，至少覆盖 functional need、lived scene、motive codes、perceivable outcome、品牌输入到用户表达的翻译链、逐页角色、素材/禁忌、benchmark、`feed_preview` 尺寸与 `full_size`。
5. 若选择器返回两个合格 attention path，设置 `prototype_count >= 2`，在 `visual-prototypes.csv` 生成两个注意力路径不同且实际可查看的概念原型；只换颜色、字体、贴纸或标题不是第二概念。同一个 AP 不能为了凑两个概念重复使用。只有一个合格探索方向时最多生成一个 `prototype_only`；返回 `prototype_gap/no_eligible_card` 时停在 `brief_only` 并补采。
6. 每个原型都实际打开检查 feed 缩略图与全尺寸；记录素材真实性、首焦点、承诺可读性、层级、中文排版、选中/淘汰原因。字卡使用 `render_text_card_cover.py` 做最终文字排版，不让图像模型拼中文。只有选中方向才能扩成 `draft-assets.csv` 的逐页成品。
7. 用户整体否定两次后，把事件顺序和 hash 写入 `visual-feedback.jsonl`，目标样本、AP 模块、真实素材三类至少改变两类；不得继续微调同一失败构图。
8. 第三方原图只用于观察，不作为生成或编辑输入；长中文优先确定性排版。探索原型未实际查看时退回 `brief_only`；最终逐页文件已生成但未取得独立人工 PASS receipt 时只能是 `rendered_needs_review`。

Fast path 的真实 12 帖发布结果、完整场景矩阵和 3 人实图盲评均为 `deferred/incomplete`。它们不阻断合同与工具交付，但也不得被表述为“流量已验证”“审美提升已证明”或已完成实验。

## 9. 事实与功效证明账本

数字、比较、排名、功效、健康、收入、时效、评价、“亲测”全部入账：

| ID | 稿件原句 | 类型 | 来源/证据 | 日期与范围 | 允许表达 | 位置 | 状态 |
|---|---|---|---|---|---|---|---|
| C01 | 逐字复制主张 | 事实/功效/体验/比较 | 链接或记录 | 样本、地区、周期、版本 | 不超过证据的措辞 | 页/段 | PASS/PARTIAL/FAIL |

同时记录来源独立性、商业关系、测量口径、对照条件、适用人群、不确定性及是否需要可见引用。

主张规则：

- 单一个案只能写“我/这名用户”，不能写“大家都会”。
- 没有对照，不写“提升、降低、更有效”；测试条件不同，不下性能胜负。
- 没有完整市场口径，不写“第一、最受欢迎、全网最低”。
- 主观感受可如实写成体验，不能伪装成客观功效。
- 参数必须对应实际型号与版本；健康、法律、金融主张升级核验。
- 免责声明不能挽救正文的明确误导。

状态：PASS = 来源可追溯、范围匹配、措辞不过界；PARTIAL = 范围、授权、日期或口径缺失；FAIL = 无证据、相互矛盾、不可靠或明显外推。
PARTIAL/FAIL 主张不得进发布稿，只能补证、收窄措辞或删除。

## 10. Compliance review 与 Creative review 分开

创意不能抵消合规风险，合规也不等于内容有效；必须输出两个状态、两组问题、两组修复动作。

### 10.1 Compliance review

检查法律与平台限制、商业披露、主张证明、版权/商标/肖像/音乐许可、隐私授权、未成年人、高敏主题、健康安全及 CTA/促销机制。

- **PASS：** 无已知阻断项；证据、授权和必要标签已进入实际稿件。
- **PARTIAL：** 有可修复缺口；修复前不得发布。
- **FAIL：** 存在欺骗、侵权、高风险误导或不可接受风险；必须重构。

Compliance 只有 PASS 才能进入发布候选。

### 10.2 Creative review

检查受众场景、primary job、开头相关性、载体匹配、因果/知识逻辑、具体价值、语言自然度、品牌辨识度、承诺一致性及信息冗余。

- **PASS：** 任务清楚、结构完整、价值具体、承诺兑现。
- **PARTIAL：** 方向成立但有一至三处明确缺口；修订后复审。
- **FAIL：** 任务混乱、载体错误、承诺未兑现或主要价值依赖套话；重写。

### 10.3 输出模板

```text
Compliance：PASS / PARTIAL / FAIL
阻断项 / 修复动作：
Creative：PASS / PARTIAL / FAIL
缺口 / 修复动作：
最终状态：可发布 / 修复后复审 / 不发布
```

两项均 PASS，最终状态才可为“可发布”。

## 11. 自然准实验与生命周期

### 11.1 自然准实验

准实验用于学习表达差异，不用于承诺算法结果；一次只改变标题角度、封面层级、开头、载体或 CTA 中一个主要变量。
尽量保持受众、主题、内容深度、商业力度和观察窗口相近；日期、热点、账号阶段均为混杂，必须记录。

```text
学习问题 / 基准版本 / 实验版本：
唯一主要变量：
primary metric / 护栏指标：
观察窗口 / 已知混杂：
样本不足条件 / 停止或回滚条件：
```

指标跟随 primary job：解释看完读与有效收藏；信任看高质量质询与资料访问；考虑看对比与真实意向；行动看合规转化并监控取消、投诉、误解。
禁止购买互动、组织虚假评论、冒充反馈或用小号制造社会证明；一次高曝光不能证明因果。

### 11.2 生命周期

| 阶段 | 触发条件 | 动作 | 判断 |
|---|---|---|---|
| 发布前 | 证据、资格与双审查完成 | 核对实际素材、标签、CTA和披露 | 是否满足发布门槛 |
| 发布一致性 | 前台内容与后台字段首次可查看 | 检查展示、组件、错字和披露位置 | 实际发布是否等于审查稿 |
| 早期诊断 | 首批有效消费或反馈出现 | 记录误解、质疑及漏斗最先掉落层 | 是否需立即纠错或停止 |
| 主观察 | 主指标达到预注册的可解释事件量，或生命周期推导的窗口结束 | 与匹配基线及护栏比较 | 保持、迭代还是判 `inconclusive` |
| 学习更新 | 共性问题、反例或新需求重复出现 | 补证据、更新选题卡、下一篇只改一个变量 | 新信号能否支持续篇 |
| 后期维护 | 规则、来源、商品事实变化，或内容仍持续获得相关消费 | 更新、复用或退役 | 内容是否仍准确、相关、合规 |

具体时间点只能由本账号数据可得性、历史分布、primary job、内容生命周期和预注册实验推导；没有这些输入就保留触发条件，不把阶段换成固定日期。
只用“保持、迭代、扩展、更新、退役”描述决策，不用“爆了/扑了”代替诊断。

## 12. 语气样本：去营销号与 AI 腔

目标语气：具体、克制、平视、有判断、口语但不随意；证据有多大，结论就写多大。

除非品牌语气有证据，删除：“家人们谁懂啊、不允许还有人不知道、闭眼入、冲就完了、狠狠拿捏、直接封神、颠覆认知、全网最全、天花板、答应我一定要试”。

坏：
> 姐妹们，这个方法真的绝了，照着做直接拿捏所有场合！

好：
> 如果你每天通勤超过一小时，先看肩带受力和空包重量；这两项比隔层数量更影响长期使用。

AI 腔高风险模式：机械“首先其次最后”、同义复述、堆叠“不是而是”、无证据的“值得注意/不难发现”、强行金句、伪造感官细节、大量引号、突然上升到人生或时代。

坏：
> 真正的高级感，从来不是价格，而是你对生活方式的深刻理解。

好：
> 预算相同时，先比较每天会用到的功能；不常用的装饰不必付费。

坏：
> 这不仅是一款产品，更是一种全新的生活态度。

好：
> 它解决桌面线材收纳，不会改善桌面尺寸，也不适合经常移动工位的人。

终稿校正三遍：事实遍圈出并入账所有主张；人话遍朗读并删口号与复述；任务遍删除不服务 primary job 的金句、细节、互动和商品信息。
每段至少承担推进、解释、证明、比较、示范、收束之一；无功能即删除。

## 13. 最终红线与自查

以下念头都表示 STOP：“先写得像真人再补授权”“大家都这样写”“只是暗示不算虚假”“标题先夸张正文再解释”“用户要爆款所以必须给公式”“参数是真的所以可写亲测”“评论反馈可以自己模拟”。

交付前逐项确认：

- [ ] 有可追溯的 observed/calculated 证据，结论未超过范围，选题卡与六个硬门槛通过。
- [ ] primary job 唯一，漏斗明确，载体由信息任务决定。
- [ ] ready 稿已绑定合格风格规则；`needs_style_research` 仅产出有机制/反例/真实素材绑定的 `candidate_only`，`qualified_cells=0` 未启用 starter。
- [ ] 故事、聊天或轮播通过对应专门门槛。
- [ ] 真实性、商业标签进入实际稿件，授权和匿名化完整。
- [ ] 有 2–3 个标题/封面版本，无流量保证。
- [ ] 标题、封面、正文、CTA 兑现同一个承诺。
- [ ] 完整正文无占位符，页面/镜头与视觉指令可执行。
- [ ] 视觉 brief、双概念原型、feed/full review、反馈/reset 与逐页成品链可追溯；未把 deferred 评测写成已完成。
- [ ] 所有事实和功效主张入账，PARTIAL/FAIL 已补证、收窄或删除。
- [ ] Compliance review 独立完成且为 PASS。
- [ ] 成人敏感商业内容的生产门禁 receipt 为当前 pass；否则稿件保持 blocked。
- [ ] Creative review 独立完成且为 PASS。
- [ ] 准实验只改一个主要变量，生命周期与停止条件明确。
- [ ] 已完成事实遍、人话遍、任务遍。
- [ ] 不含伪真人、假测评、虚假社会证明或固定爆款公式。

任何一项未通过，不得宣称“生产级”“可直接发布”或“审查完成”。
