# 用户画像分析 Skill

## 1. Skill名称
用户画像分析（半醒之间版）

## 2. 使用场景
- 对齐「半醒之间」核心用户画像 → [[02_用户研究/核心用户画像]]
- 内容策略调整时的用户验证
- 新产品/服务设计前的用户研究（远期）
- 每月用户画像更新 → 更新 [[memory/audience_memory]]

## 3. 输入格式
```yaml
data_sources:
  - comment_samples: []
  - competitor_audience: ""
  - search_keywords: []
  - interaction_data: {}
focus_area: ""
```

## 4. 执行流程
1. **数据收集**：从评论、私信、竞品关注者中收集信息
2. **人口属性分析**：年龄、职业、城市、收入
3. **行为特征分析**：阅读习惯、消费偏好、信息获取渠道
4. **心理特征分析**：价值观、动机、恐惧、渴望
5. **痛点分级**：核心痛点→次要痛点→隐性需求
6. **画像建模**：创建2-3个典型用户角色
7. **内容映射**：每个画像匹配对应的内容方向

## 5. 输出格式
```markdown
## 用户角色：{角色名称}
- 年龄/职业：{基本信息}
- 一句话描述：{角色特征}
- 核心痛点：{top 3}
- 内容偏好：{喜欢什么类型的内容}
- 决策路径：{从看到到行动的过程}
- 触发词：{最能引起ta注意的词}
```

## 6. 示例
```markdown
## 用户角色：焦虑的职场进阶者
- 年龄/职业：28岁，互联网运营
- 一句话描述：工作3-5年，想突破但找不到方向
- 核心痛点：
  1. 感觉在重复劳动，没有成长
  2. 想学习但不知道学什么
  3. 看到同龄人成功更焦虑
- 内容偏好：方法论+真实案例+可执行步骤
- 决策路径：看到标题→快速浏览→收藏→周末细看→尝试
- 触发词：瓶颈、突破、系统、可执行、真实
```
