# 用户画像分析

## 使用场景

- 对齐 [[02_用户研究/核心用户画像.md|半醒之间核心用户画像]]
- 内容策略调整时的用户验证
- 新产品/服务设计前的用户研究（远期）
- 每月用户画像更新 → 更新 [[memory/audience_memory|audience_memory]]

## 输入格式

### 必须输入
| 字段 | 类型 | 说明 |
|------|------|------|
| data_sources | object | 数据来源 |
| focus_area | string | 分析重点 |

### 可选输入
| 字段 | 类型 | 说明 |
|------|------|------|
| data_sources.comment_samples | string[] | 评论样本 |
| data_sources.competitor_audience | string | 竞品受众分析 |
| data_sources.search_keywords | string[] | 搜索关键词 |
| data_sources.interaction_data | object | 互动数据 |

## 执行流程

1. **数据收集**：从评论、私信、竞品关注者中收集信息
2. **人口属性分析**：年龄、职业、城市、收入
3. **行为特征分析**：阅读习惯、消费偏好、信息获取渠道
4. **心理特征分析**：价值观、动机、恐惧、渴望
5. **痛点分级**：核心痛点 → 次要痛点 → 隐性需求
6. **画像建模**：创建 2-3 个典型用户角色
7. **内容映射**：每个画像匹配对应的内容方向

### 判断标准（if-then）
- 如果连续两个月新增用户画像特征变化 ≥20%，则必须更新画像。
- 如果画像与 [[02_用户研究/核心用户画像.md|核心用户画像]] 偏离 ≥30%，则必须上报 CEO Agent。
- 如果某个用户角色无法对应到具体选题，则必须重新评估该角色有效性。

## 输出格式

### 必须字段
| 字段 | 类型 | 说明 |
|------|------|------|
| 角色名称 | string | — |
| 年龄/职业 | string | 基本信息 |
| 一句话描述 | string | 角色特征 |
| 核心痛点 | string[] | top 3 |
| 内容偏好 | string | 喜欢什么类型内容 |
| 决策路径 | string | 从看到到行动的过程 |
| 触发词 | string[] | 最能引起注意的词 |

### 输出示例
```markdown
## 用户角色：{角色名称}
- 年龄/职业：{基本信息}
- 一句话描述：{角色特征}
- 核心痛点：{top 3}
- 内容偏好：{喜欢什么类型的内容}
- 决策路径：{从看到到行动的过程}
- 触发词：{最能引起ta注意的词}
```

## 示例

### 示例 1：焦虑的职场进阶者
**输入**：
```yaml
data_sources:
  comment_samples:
    - "工作3年了，感觉在重复劳动"
    - "想学但不知道学什么"
    - "看到同龄人成功更焦虑"
  search_keywords: ["职业瓶颈", "成长方向", "30岁焦虑"]
focus_area: "内容策略调整"
```

**输出**：
```markdown
## 用户角色：焦虑的职场进阶者
- 年龄/职业：28 岁，互联网运营
- 一句话描述：工作 3-5 年，想突破但找不到方向
- 核心痛点：
  1. 感觉在重复劳动，没有成长
  2. 想学习但不知道学什么
  3. 看到同龄人成功更焦虑
- 内容偏好：方法论 + 真实案例 + 可执行步骤
- 决策路径：看到标题 → 快速浏览 → 收藏 → 周末细看 → 尝试
- 触发词：瓶颈、突破、系统、可执行、真实
```
