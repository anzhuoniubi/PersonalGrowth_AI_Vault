# Content Style - 写作风格指南

## 核心表达原则

### 1. 朋友式对话
- 用"你"而不是"大家"
- 像朋友聊天一样自然，而不是老师讲课
- 偶尔自嘲，增加亲近感
- 使用口语化但不失专业的语言

### 2. 真实具体
- 每个观点都要有亲身经历支撑
- 数据、数字、具体场景代替模糊描述
- 好的： "我连续31天早上6点起床后..."
- 不好的： "早起很重要"

### 3. 结构化清晰
- 短段落（不超过3行）
- 多用小标题和列表
- 开头用钩子，中间有节奏变化
- 结尾有明确的行动号召

## 小红书特有风格

### 标题公式
```
数字 + 结果承诺/痛点 + 情绪词
```
示例："25岁到30岁，我做对了这3件事改变人生"

### 开头钩子
- 痛点共鸣型："你是不是也觉得..."
- 结果前置型："从月薪8k到25k，我只用了..."
- 认知反转型："关于自律，90%的人都理解错了"
- 故事引入型："上个月我做了一个疯狂的决定..."

### 正文结构
1. **共情开头**：理解读者现状
2. **转折**：我曾经的困境
3. **方法/认知**：具体怎么做
4. **案例佐证**：我的实际效果
5. **总结升华**：金句+行动号召

### 金句风格
- 简短有力（15字以内最佳）
- 有反差感
- 易于传播
- 使用修辞（比喻、排比、对仗）

示例：
- "自律不是目的，自由才是。"
- "慢下来，才走得远。"
- "成长不是变完美，是变自洽。"

## 视觉语言
- 封面：干净留白，主标题突出
- 配色：低饱和度暖色调（米白+焦糖+深灰）
- 图片：实拍＞插画＞网图
- 排版：行间距宽松，重点加粗

## 个人IP风格锚定
- 语气：温和但有力量
- 身份：先行一步的同行者，不是导师、不是专家、不是成功者
- 态度：实验者心态（"我在探索，不是我在教"）
- 价值观：真实、系统、可持续、长期主义
- **IP定位**：[[01_个人定位/半醒之间IP定位]]
- **表达铁律**：先问"这个观点如何帮助一个明天还要上班的人？"，回答不了就先不发

### 三个内容层的语气差异
| 层级 | 语气 | 表达方式 |
|------|------|---------|
| 觉察层 | 温柔共鸣 | "你是否也感觉到……" |
| 重构层 | 理性启发 | "如果我们换一个角度看……" |
| 创造层 | 真实记录 | "这是我正在做的第X天……" |

## 公众号特有风格

### 定位差异
- 小红书：情绪共鸣×身份标签（轻、短、快）
- **公众号：深度系统×长阅读（长、深、粘）**
- 同一条认知，小红书给"启发"，公众号给"框架"

### 标题公式
```
认知反差 + 价值承诺（不含数字）
```
示例：
- "内耗不是病，是未被启用的认知复利"
- "你的系统不需要更复杂，只需要更自洽"

### 正文结构
1. **故事/场景引入**（300-500字，建立代入感）
2. **问题拆解**（3层递进分析）
3. **方法框架**（可保存的认知模型）
4. **实践验证**（数据/时间跨度）
5. **开放结尾**（引发读者留言讨论）

### 格式规范
- 段落稍长（3-5行），不碎片化
- 可适度使用引用 `> ` 语法和分割线 `---`
- 文末加「推荐阅读」引导往期文章
- 阅读时长标记：如"本文约3000字，阅读需7分钟"

### 排版
- 字号15-16px，行间距1.75
- 全文配色不超过3种
- 重点句加粗，非变色/下划线
- 配图间隔≤500字

## 播客（音频/视频）风格

### 内容定位
- 不做"知识播客"，做 **"陪伴式思考播客"**
- 主题：单期拆解一个成长困惑，给出实操建议
- 长度：20-35min / 期

### 结构模板
1. **开场**（2min）：上周最触动我的一个瞬间
2. **主体**（15-25min）：一个认知 + 一个故事 + 一个方法
3. **反刍**（3-5min）：我试过之后的变化
4. **结尾**（2min）：下一期的预告 + 行动邀请

### 语气
- 保持口语化，像在咖啡馆聊天
- 允许沉默和停顿（思考的感觉）
- 避免「今天我们来聊聊……」的学术开场

## AI辅助内容政策

### 使用原则
1. **AI是协作者，不是代笔人**：所有发布内容必须经过人工修改和审定
2. **AI负责提效，人负责灵魂**：
   - AI可做：素材整理、标题生成、结构建议、敏感词扫描
   - 人必须做：价值判断、个人故事、情感温度、最终决定
3. **透明度**：
   - AI辅助选题/修改框架 → 无需声明（行业惯例）
   - 纯AI生成后人工润色 → 需在创作笔记中标注，不冒充纯原创
   - 严格禁止：用AI写个人经历/感受/观点后假装亲身经历

### 质量控制
- AI生成初稿后，必须经过**黄金三角质检**（见内容生产库模板）
- 敏感词扫描是AI强项，但红线判断由创作者负责
- 每次发布后，将实际数据反馈至对应skill的JSONL以优化模型

## 编辑节奏

### 周度节奏
| 星期 | 动作 | 产出 |
|:----:|------|------|
| 一 | 选题会 | 下1-2篇选题卡片 |
| 二-三 | 内容创作 | 正文+封面 |
| 四 | 排期发布 | 1-2篇小红书 |
| 五-六 | 数据追踪+复盘 | 更新追踪表 |
| 日 | 周度复盘 | 10_数据复盘/周报 |

### 发布密度
- **小红书**：每周2-3篇（不堆量，质量优先）
- **公众号**：每月2-4篇（深度长文，与小红书主题联动）
- **播客**：每月2期（筹备期，正式启动待定）
